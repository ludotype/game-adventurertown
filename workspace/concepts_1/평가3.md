# 컨셉 3 평가: 던전 관리 + 외교 시뮬레이션 게임

## 평가 기준
사용자의 조건: Papers Please급 참신함, 귀여운 여자애들+재미있는 대사, 아티스트+시나리오 라이터 혼자 2개월 완성, 비주얼 노벨 엔진 연계, 최소 프로그래밍

## 1. 참신함 / 인터랙티브함: A
- 던전 관리 + 연애 시뮬레이션이라는 장르 조합 자체가 매우 독특하고 흥미로움
- 외교/내정/전투/연애가 복합적으로 어우러지는 구조는 깊이 있는 gameplay를 약속함
- "기억을 잃은 주인공이 용족 프린세스의 부하가 된다"는 설정도 매력적임

## 2. 귀여운 여자애들 / 재미있는 대사 적합성: A
- 메이드, 용족 프린세스, 메이드장, 붙잡힌 기사 등 **"귀여운 여자애들"이 자연스럽게 등장**하는 구조
- 사용자의 아티스트 강점을 100% 발휘할 수 있음
- 다양한 캐릭터와의 연애 이벤트, 스토리 이벤트가 풍부하게 나올 수 있음

## 3. 2개월 혼자 완성 가능성: D
- 던전 개축/증축 시스템, 몬스터 소환/배치, 외교 시스템, 내정 관리, 세력 규모 계산, 전투 시스템, 연애 시뮬레이션...
- **시스템이 너무 많음**. 하나하나가 각각 하나의 게임 수준의 복잡도
- 프로그래머가 있었다면 몰라도, **혼자 2개월 내에는 불가능에 가까움**
- 시스템 간 밸런스 조정만 해도 수 주일이 걸림

## 4. 비주얼 노벨 엔진 연계성: C
- 던전 빌딩은 그리드/맵 기반 UI가 필수적임. VN 엔진의 대화 중심 구조와 궁합이 좋지 않음
- 몬스터 배치, 외교 협상, 내정 관리 등도 모두 UI 중심의 관리 시스템
- VN 엔진에서 이를 모두 구현하려면 엔진을 거의 재구축해야 함

## 5. 최소 프로그래밍 필요도: 매우 높음
- 던전 빌딩, 몬스터 AI/배치, 외교 수치 시뮬, 세력 규모 계산, 엔딩 분기 조건 등 압도적 프로그래밍량

## 종합 평가
- **컨셉 자체는 5개 중에서 가장 흥미롭고 사용자의 강점과도 잘 맞음**
- 그러나 **"프로그래머가 날 버리고 가서 나 혼자서 2개월 내에 완성해야 한다"**는 현실 조건을 완전히 무시함
- 이 컨셉은 팀 프로젝트 또는 6개월~1년 이상의 개발 기간이 필요함
- **단기간 추천하지 않음**: 현실적인 조건과 맞지 않음. 나중에 팀을 꾸려서 도전할 프로젝트로 보관할 것
