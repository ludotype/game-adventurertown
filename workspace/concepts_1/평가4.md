# 컨셉 4 평가: 프린세스 메이커 클론 (UI 경량화)

## 평가 기준
사용자의 조건: Papers Please급 참신함, 귀여운 여자애들+재미있는 대사, 아티스트+시나리오 라이터 혼자 2개월 완성, 비주얼 노벨 엔진 연계, 최소 프로그래밍

## 1. 참신함 / 인터랙티브함: B
- 프린세스 메이커는 잘 알려진 장르이므로 장르 자체의 참신함은 낮음
- 그러나 **"어려진 여마왕/드래곤을 키운다"**는 설정은 기존 프린세스 메이커와 차별화됨
- Papers Please만큼의 참신함은 아니지만, **"육성 시뮬레이션 + 비주얼 노벨"**이라는 조합은 효과적임
- 매 턴의 스케줄 선택이 곧 인터랙션이며, 이 선택이 누적되어 엔딩에 영향을 줌

## 2. 귀여운 여자애들 / 재미있는 대사 적합성: A
- **메인 캐릭터(여마왕/드래곤)가 중심**에 서 있으므로 사용자의 아티스트 강점을 100% 발휘 가능
- 성장하는 과정에서 다양한 연애 이벤트, 일상 이벤트, 특수 이벤트 등 대사량이 풍부함
- NPC 수를 줄인다고 했으므로, 적은 수의 NPC와 깊이 있는 관계를 만들 수 있음
- 시나리오 라이터로서의 강점을 극대화할 수 있는 구조

## 3. 2개월 혼자 완성 가능성: A
- UI를 최소화하고 NPC 수를 줄인다면 현실적인 규모가 됨
- 비주얼 노벨 엔진과 완벽하게 호환되므로 엔진 커스터마이징이 거의 불필요함
- 핵심 로직은 단순함: 능력치 변수 + 스케줄 선택지 + 이벤트 분기
- 2개월이면 기본 육성 시스템 + 스토리 + 아트를 모두 커버할 수 있는 규모

## 4. 비주얼 노벨 엔진 연계성: A
- 스케줄 선택지 + 시간 경과 + 이벤트는 VN 엔진의 기본 구조와 동일함
- 기존에 완성된 비주얼 노벨 스타일 라이브러리를 거의 그대로 활용 가능
- 추가로 필요한 것은 능력치 패널과 간단한 스케줄 UI 정도

## 5. 최소 프로그래밍 필요도: 낮음~중간
- 능력치 시스템, 스케줄-이벤트 매핑, 엔딩 분기 조건 정도만 필요함
- VN 엔진 기반으로는 비교적 간단한 추가 작업

## 종합 평가
- **가장 현실적이고 안전한 선택**
- 사용자의 **모든 핵심 강점(아티스트, 시나리오 라이터)**과 완벽하게 부합
- 2개월 내 혼자 완성이 실제로 가능한 규모
- 다만 **"Papers Please만큼 참신한가?"**라는 질문에 대한 답은 "장르는 덜 참신하지만, 설정과 스토리로 커버 가능"이라는 수준
- **강력 추천**: 단기간 인디 게임 개발에 최적화된 컨셉
