## 1. 사용자의 상황 및 요구사항

### 현재 상황
- 프로그래머가 팀을 떠나 **혼자서 개발**해야 함
- 사용자 본인은 **아티스트 + 시나리오 라이터**
- 완성 기한: **2개월 내**
- 기존에 **비주얼 노벨 스타일 게임플레이 라이브러리는 완성**되어 있음
- **기존 픽셀 아트 몬스터 스프라이트** 다수 보유 (픽셀 아티스트가 제작)

### 핵심 요구사항
1. **Papers, Please** 수준의 **참신하고 인터랙티브한 방식**
2. **제작 자원과 비용이 별로 들지 않는 인디 게임**
3. **귀여운 여자애들이 많이 나오고 재미있는 대사**가 들어 있을 것
4. 기존 **비주얼 노벨 엔진과 자연스럽게 연계**될 것
5. **최소한의 프로그래밍**으로 구현 가능할 것

## 2. 5개 원본 컨셉 요약

### 컨셉 1: 비무장지대 감시기지 벙커 게임
전체주의 제국의 최전방 감시 벙커에서 근무하며, 방어 시설 점검/식사/감시/보고/도덕적 선택을 하는 Papers Please / This War of Mine류 게임.

### 컨셉 2: 모험가 길드 관리 게임
1인칭 길드 마스터가 되어 모험가들에게 퀘스트를 부여하고 장비를 분배하는 매니지먼트 게임. 전투는 뉴스 틱커+그래픽 카드로 간접 진행. 귀환 후 전리품 판매 및 마을 투자.

### 컨셉 3: 던전 관리 + 외교 시뮬레이션
기억을 잃은 주인공이 용족 프린세스의 부하가 되어 던전을 개축/증축하고 몬스터를 배치하며 외교/내정/연애를 펼치는 게임. 세력 규모에 따른 다양한 엔딩.

### 컨셉 4: 프린세스 메이커 클론 (UI 경량화)
어려진 여마왕/드래곤을 키우는 프린세스 메이커 스타일 육성 시뮬레이션. UI 최소화, NPC 수 축소, 스토리와 메인 캐릭터에 집중.

### 컨셉 5: Idle 농장 운영 + 연애 게임
방침만 정해주는 idle 타입 농장 게임. 스케줄 지정으로 진행되며 외출을 통해 NPC와 연애. 비주얼 노벨 + 간단한 UI.

## 4. 사용자의 조합 제안 및 평가

### 제안 1: 컨셉 1 (벙커) + 컨셉 5 (연애)

#### 조합 구조
전쟁 벙커에서 idle 스케줄(식사/감시/여가)을 짜며 동료들과 연애하고, 동시에 전선 감시 및 도덕적 선택을 내리는 게임.

#### 평가
- **긍정적**: 벙커라는 폐쇄 공간이 오히려 연애 시뮬에 유리 (좁은 공간+소수 인원=긴밀한 관계). Papers Please식 도덕 선택이 연애와 연동되면 서사적 임팩트가 클 수 있음.
- **부정적**: 전쟁의 무거운 분위기와 귀여운 아트/가벼운 대사가 **근본적으로 충돌**. 벙커에 귀여운 여캐가 많다는 설정도 현실성이 약함. 플레이어가 "즐거운 건지 슬픈 건지" 혼란스러울 수 있음. 프로그래밍도 복잡해짐.
- **종합**: 컨셉적으로 흥미롭지만, **사용자의 핵심 강점과 방향성이 맞지 않음**. 영화/소설로는 성공할 수 있으나 게임으로는 감정 혼란 위험이 큼. **보류**.
- **만약 설정을 바꾼다면 가능성 있음**: 전쟁 벙커 대신 "우주 정거장", "초능력 학원 방범 벙커", "재난 대피소" 등으로 가벼운 폐쇄 공간으로 변경하면 Papers Please식 도덕 선택 + 폐쇄 공간 연애라는 참신한 조합이 될 수 있음.

### 제안 2: 컨셉 2 (길드 관리) + 컨셉 4 (프린세스 메이커)

#### 조합 구조
길드 마스터가 길드를 운영하면서 특정 길드원/수련생을 직접 키우는 육성 시뮬레이션. 길드 운영은 간접적, 육성은 직접적 진행.

#### 평가
- **긍정적**: 컨셉 2의 치명적 약점이었던 **"스토리 방향 불명확"**을 컨셉 4의 "육성 목표"가 완벽하게 해결. "내가 키우는 아이에게 최고의 장비와 경험을 주려고 길드를 돌린다"는 감정적 동기가 생김. 기존 픽셀 아트도 활용 가능.
- **부정적**: 두 시스템의 시간 흐름 통합 문제 ("1개월=1턴" 동안 길드 운영은 어떤 단위로?). UI 복잡도 증가 (육성 패널+길드 관리 패널). 집중도 분산 위험.
- **종합**: 이 조합의 현실화를 위해서는 **"길드 관리를 대폭 축소"**해야 함. 퀘스트 결과를 텍스트 이벤트로 처리하고 경제/전투 시뮬/업그레이드를 제거. 길드원 3~5명으로 제한하여 각자를 연애/이벤트 대상으로. **이렇게 축소하면 결국 "컨셉 4 단독 + 길드원 NPC 연애 이벤트 추가" 형태로 수렴함.**

## 5. 컨셉 2 재평가: "시청각 모험 시뮬레이션"

### 사용자의 실제 비전 (오해 바로잡기)
사용자가 생각한 "뉴스 틱커"는 단순 텍스트가 아니라:
- **UI 위에서 꼬물꼬물 움직이는 치비 모험가 스프라이트**
- **실시간(또는 턴제)으로 펼쳐지는 모험/전투 시뮬레이션**
- **성격 특성(trait) 기반 템플릿 대사 시스템**: 특성이 대사와 행동(전투 판정)에 직접 반영됨
- **수백 개의 상황-특성 조합 템플릿 메시지** (미리 작성된 대사풀)
- **비주얼 노벨 메시지 창에 띄우는 것이 아닌**, 게임 화면(UI 맵 위)에서 상황과 함께 자연스럽게 발생하는 이벤트

예시 상황:
> A가 고블린에게 맞았다! 많이 아프다!
> B: "A! 괜찮아? 후퇴할까?"
> 하지만 A는 '자존심이 강함' 특성을 갖고 있음.
> A: "괜찮아! 후퇴는 남자답지 않으니까."

### 재평가 결과
- **참신함**: A ("관리자가 지켜보는 시뮬레이션" + 특성 기반 템플릿 대사 + 치비 스프라이트라는 조합이 독특함)
- **강점 발휘**: A (기존 픽셀 아트 활용 + 수백 개 템플릿 대사 = 사용자의 아트+시나리오 강점 100% 발휘)
- **2개월 완성**: B (빡빡하지만 가능. 단, 스코프를 엄격히 제한해야 함)
- **VN 엔진 연계**: B (관리 페이즈는 VN 구조와 잘 맞고, 모험 페이즈는 VN 엔진 위에 오버레이되는 미니 게임으로 구현 가능)
- **프로그래밍**: 중간~높음 (치비 이동 + 조우 + 전투 수치 + 템플릿 출력)

### 이 컨셉의 진짜 강점
1. **"보는 재미"가 있는 관리 게임**: 플레이어는 결과만 받는 게 아니라, 파견한 모험가들이 살아 움직이는 것을 지켜본다. Majesty/Dwarf Fortress의 "간접 관찰의 몰입도"를 비주얼 노벨 형태로 구현.
2. **성격 특성 기반 대사 시스템**: 특성이 실제 전투 행동과 대사에 반영되면 캐릭터가 살아있게 느껴짐.
3. **템플릿 기반 접근의 현명함**: 수백 개의 대사를 모두 하드코딩하지 않고 (상황×특성) 매트릭스로 관리하면 콘텐츠 양은 많지만 시스템은 단순.

### 현실적 제약 및 MVP 스코프
2개월 내 혼자 완성하려면:
- **모험가**: 3명 (너무 많으면 템플릿 대사 폭발)
- **몬스터**: 5종 정도 (기존 픽셀 아트 활용)
- **맵**: 1개 던전/지역
- **템플릿**: 100~200개 내외로 MVP 시작
- **특성**: 5~8개

| 항목 | 컨셉 4 (프린세스 메이커) | 재평가 컨셉 2 (시청각 모험 시뮬) |
|------|--------------------------|-----------------------------------|
| 참신함 | B | **A** |
| 강점 발휘 (아트+대사) | A | **A** |
| 2개월 완성 | A (안정적) | **B (빡빡하지만 가능)** |
| 프로그래밍 | 낮음~중간 | **중간~높음** |
| VN 엔진 연계 | A | **B** |
| 시스템 복잡도 | 낮음 | **중간** |

## 6. 템플릿 대사 시스템 상세 설계

### 시스템 개요
각 모험가는 고유한 **성격 특성(trait)** 1~3개를 보유. 모험 중 발생하는 **상황(situation)**에 따라, 보유 특성 중 하나가 랜덤 또는 우선순위에 따라 발동하여 **템플릿 대사**가 출력됨.

### 데이터 구조 예시
```json
{
  "situation": "전투_피격_중상",
  "templates": {
    "자존심강함": "괜찮아! 후퇴는 남자답지 않으니까.",
    "겁쟁이": "아얏... 이, 이 정도는 아무것도 아냐... (발이 떨림)",
    "낙천적": "아프지만, 이것도 추억이겠지! ...아파.",
    "냉정함": "...피가 너무 나. 즉시 치료가 필요해.",
    "거만함": "흥, 이딴 걸로 쓰러질 내가 아니야!",
    "동료애": "(다른 동료를 보며) ...괜찮아. 난 아직 서 있을 수 있어.",
    "무모함": "오히려 좋아! 피가 끓어오른다!"
  }
}
```

### 상황 카테고리
1. **전투_시작**: 몬스터와 조우했을 때
2. **전투_피격_경상**: 가벼운 피해를 입었을 때
3. **전투_피격_중상**: 심각한 피해를 입었을 때
4. **전투_동료_위험**: 동료가 위기에 처했을 때
5. **전투_동료_구출**: 동료를 구했을 때
6. **전투_승리**: 전투에서 이겼을 때
7. **전투_패배_후퇴**: 후퇴해야 할 때
8. **전투_발견_보물**: 전리품/보물을 발견했을 때
9. **전투_발견_함정**: 함정을 발견했을 때
10. **탐험_갈림길**: 갈림길에서 선택해야 할 때
11. **탐험_수상한_소리**: 수상한 소리를 들었을 때
12. **탐험_휴식**: 휴식을 취할 때
13. **탐험_동료_대화**: 동료와 대화할 때
14. **탐험_배고픔**: 식량이 부족할 때

### 최종 추천

#### 주 추천: 컨셉 4 (프린세스 메이커 클론, UI 경량화)
**이유**: 사용자의 **모든 현실 조건**과 가장 잘 맞습니다. 2개월 내 혼자 완성이 실제로 가능하며, 비주얼 노벨 엔진을 거의 그대로 활용할 수 있습니다. 프로그래밍이 거의 필요 없고, 시나리오 라이터로서의 강점을 극대화할 수 있습니다.

#### 대안 추천 1: 재평가된 컨셉 2 (시청각 모험 시뮬레이션)
**이유**: **5개 컨셉 중에서 가장 참신하고 기억에 남는 게임**이 될 수 있습니다. 기존 픽셀 아트를 활용하고, 특성 기반 템플릿 대사 시스템은 시나리오 라이터로서의 강점을 100% 발휘할 수 있는 구조입니다.
**단, 조건**: 프로그래밍에 어느 정도 자신이 있어야 하며, MVP 스코프(모험가 3명, 몬스터 5종, 템플릿 100~200개)를 엄격히 지켜야 합니다.

#### 대안 추천 2: "컨셉 4 + 재평가 컨셉 2의 하이브리드"
**구조**: 프린세스 메이커 스타일의 육성 시스템을 기반으로, "외출/모험" 선택지를 골랐을 때만 치비 스프라이트 시뮬레이션이 펼쳐지는 구조.
**장점**: 안정적인 육성 시스템(컨셉 4)에 참신한 모험 시뮬(재평가 컨셉 2)을 **선택적으로 추가**하여, 프로그래밍 부담을 줄이면서도 차별화를 꾀할 수 있습니다.
