# 컨셉 2 평가: 모험가 길드 관리 게임

## 평가 기준
사용자의 조건: Papers Please급 참신함, 귀여운 여자애들+재미있는 대사, 아티스트+시나리오 라이터 혼자 2개월 완성, 비주얼 노벨 엔진 연계, 최소 프로그래밍

## 1. 참신함 / 인터랙티브함: C
- 길드 관리 게임은 이미 많이 존재하는 장르 (Guildmaster Story, Dungeon Village, Tavern Master 등)
- 뉴스 틱커+그래픽 카드로 전투를 간접적으로 보여주는 방식은 참신할 수 있으나, 이 방식 자체가 지루하게 느껴질 위험이 큼
- 플레이어가 직접 전투에 참여하지 않으므로 긴박감이 떨어짐

## 2. 귀여운 여자애들 / 재미있는 대사 적합성: B
- 길드에 등록된 여성 모험가들을 귀엽게 그릴 수는 있음 (기존 픽셀 아트 활용 가능)
- 길드 마스터라는 포지션이 NPC와의 직접 대화보다는 "관리"에 가까워서 깊이 있는 대사가 나오기 어려움
- 모험가들의 개성을 살리려면 시나리오가 많이 필요하지만, "어떤 스토리를 짜야 할지 전혀 감이 오지 않음"이라는 사용자의 피드백이 치명적임

## 3. 2개월 혼자 완성 가능성: C
- 기존 픽셀 아트 스프라이트가 있다는 것은 큰 장점
- 그러나 경제 시뮬레이션(시세, 판매, 수익), 전투 시뮬레이션(모험가 능력치 대비 몬스터), 마을 업그레이드 시스템 등 프로그래밍량이 많음
- 시스템 프로그래밍 + 시나리오 + 아트를 혼자 2개월 내에 하기에는 과부하

## 4. 비주얼 노벨 엔진 연계성: C
- 관리 시뮬레이션의 핵심은 UI 패널, 수치, 아이템 슬롯, 퀘스트 보드 등인데, 이는 VN 엔진의 대화 중심 구조와 자연스럽게 맞지 않음
- VN 엔진 위에 얹으려면 상당한 UI 커스터마이징이 필요함

## 5. 최소 프로그래밍 필요도: 높음
- 퀘스트 할당, 장비 분배, 전투 시뮬레이션, 경제 시스템, 업그레이드 트리 등 상당한 프로그래밍 필요

## 종합 평가
- 사용자가 직접 **"어떤 스토리를 짜야 할지 전혀 감이 오지 않음"**이라고 언급한 것이 가장 큰 약점
- 시스템도 복잡하고 스토리 방향성도 불명확하여 2개월 내에 완성하기 어려움
- 기존 픽셀 아트가 있음에도 불구하고, 사용자의 시나리오 라이터로서의 강점을 발휘하기 어려운 구조
- **추천하지 않음**: 시스템 부담이 크고 스토리 방향이 불분명함
