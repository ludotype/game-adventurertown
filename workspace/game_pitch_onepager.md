# Guild Master — 게임 컨셉 한 장 요약

## 한 줄 소개
**위험지대 마을에서 모험가로 살아가며, 한 사람의 마음을 얻는 1인칭 판타지 연애 어드벤처.**

---

## 핵심 루프 (하루 7~16분)

```
[마을 일상: 대화/행동] → [던전 탐험: 텍스트 어드벤처] → [귀환/결산] → [주점에서의 시간] → [반복]
```

- **24시간제 시간 진행**: 행동마다 시간 소비. 취침 시 다음 날 아침.
- **장소 이동**: 여관 → 복도 → 로비 → 거리 → 광장 → 주점/상점 (노드 기반)
- **NPC 스케줄**: 시간대별로 NPC가 다른 장소에 등장. 조우 후 대화/상호작용.

---

## 3가지 기둥

### 1. 관계 (Relationship)
- 단일 MVP 히로인 **루이제** (여관 주인)를 시작으로, 단계적 호감 상승.
- `무관심 → 관심 → 친밀`. 호감도에 따라 대사/이벤트/상호작용 분기.
- MetricStore 기반 (`npc.luise.affection`). 동일 시스템으로 추가 캐릭터 확장 예정.

### 2. 탐험 (Adventure)
- **텍스트 어드벤처 북 스타일**: VN 메시지 창 + 선택지로 진행.
- HP / 정신 / 탐색도 관리. "언제 돌아갈지"가 핵심 전략.
- **전투는 최소화**: 플레이어는 영웅이 아님. 전투보다 생존 판단과 관계에 집중.

### 3. 일상 (Daily Life)
- 시간대별 NPC 위치 변화. 장소 이동과 행동 선택으로 시간 소비.
- 매일의 작은 선택이 누적되어 관계 변화. 늦은 귀환 → 루이제 걱정 이벤트 등.

## 시스템 특징

| 요소 | 방식 |
|---|---|
| **엔진** | Godot 4 + Ink 내러티브 엔진 (inklecate 컴파일, inkgd 런타임) |
| **데이터** | JSON 폴더 스캔 Auto-Discovery. 새 장소/NPC/대화/이벤트 = 파일 추가만. |
| **대화** | Ink `.ink` 파일 → `.ink.json` 컴파일 → Godot InkBalloon. 태그 기반 CG/효과음. |
| **행동** | ActionRunner 기반. sequence/if/choice/random 5개 합성 어휘로 거의 모든 이벤트 표현. |
| **아트** | 고품질 캐릭터 CG (표정 3~5종). 장소별 배경 일러스트. 2D VN 인터페이스. |

## 전투 시스템 — 현재 설계 및 질문

> **현재 방향**: 전투는 "구색만 맞추는 수준". 턴제 RPG 없이, **텍스트 선택지 + 수치 변화**로 처리.
> 예: "한 무리의 래틀링들이 보인다. [돌진한다] [숨어서 살핀다] [후퇴한다]" → 선택에 따라 HP/정신 변동.

### 외부 피드백 요청

1. **텍스트 어드벤처 북 스타일의 전투가 VN 인터페이스와 자연스럽게 통합될 수 있을까?** 메시지 창 안에서 선택지로 전투를 처리하는 방식이 몰입을 깨지 않을까?

2. **HP/정신/탐색도 3가지 수치만으로 던전 탐험의 긴장감과 전략성을 충분히 줄 수 있을까?** 더 필요한 수치나 메커닉이 있다면 무엇일까?

3. **"전투를 최소화"할 때, 플레이어가 전투/탐험 자체를 회피하려는 동기가 약해지지 않을까?** 던전 탐험의 매력을 "전투"가 아닌 다른 무엇으로 채울 수 있을까?

4. **관계 시스템과 탐험 시스템의 연결고리**로서, 귀환 시각/탐험 결과가 루이제와의 관계에 영향을 주는 정도가 적절할까? 너무 강하면 스트레스, 너무 약하면 동기 부족.
