# 📄 Guild Master — 그래픽 및 UI 스펙 문서 (v1.2 수정안)

## 1. 개요 및 디자인 철학
본 문서는 프로토타입 그래픽 및 UI 스펙입니다. **고전 텍스트 어드벤처(초기 PC-98/88)의 미니멀리즘**을 현대적으로 재해석하는 방향으로 선회했습니다. 

이는 비주얼 폴리싱을 후순위로 미루고, 핵심 게임 루프와 텍스트 내러티브 엔진(Ink)의 통합 시스템 구현을 가속화하기 위한 전략적 선택입니다.

### 핵심 강점
* **극대화된 텍스트 몰입:** 불필요한 아이콘과 버튼 테두리를 제거하여 유저가 오직 이야기와 행동 선택 텍스트에만 집중하게 만듭니다.
* **고전적 향수 자극:** 마우스를 가져가면 텍스트의 색상이 바뀌거나 밑줄이 생기는 방식의 고전적 인터페이스를 통해 스타일리시한 '올드 쿨' 감성을 전달합니다.
* **개발 효율성:** 복잡한 UI 에셋 제작 시간을 생략하고, Godot의 `Label` 및 `RichTextLabel` 노드 설정을 통해 거의 모든 UI를 즉시 구현할 수 있습니다.

---

## 2. 아트 및 해상도 스펙

### 2.1. 비주얼 스타일
* **그래픽:** 기존 결정된 스타일리시 팝 판타지(Bold-line 카툰)를 유지합니다. 하지만 UI 단계에서는 그래픽 요소의 개입을 최소화합니다.
* **UI 테두리:** `image_23.png`에 적용된 투박한 오래된 철제 질감의 베이스 프레임 테두리를 사용합니다.
* **폰트 (가장 중요):** 가독성이 좋은 고해상도 판타지풍 세리프(Serif)체 혹은 고전 타이프라이터 스타일의 폰트를 채택합니다. 모든 폰트는 네이티브 1080p에서 깨지지 않는 고해상도 렌더링을 보장해야 합니다.

### 2.2. Native 해상도 및 스케일링
* **기준 해상도:** 1920 × 1080 (FHD, 16:9)
* **Godot 셋팅:** `Stretch Mode: canvas_items`, `Aspect: expand` 유지. 텍스트 레이블이 해상도 변화에 맞춰 완벽하게 벡터 렌더링되도록 보장합니다.

## 3. 고정 프레임 UI 레이아웃 사양 (v1.2 적용)

`image_23.png`의 레이아웃 구조를 엄격히 따릅니다.

### 3.1. 화면 구성도

```

+--------------------------------------------------------+
|  [상단 표시줄 (Text-Only)]                               |
|  Funds, Health, Mental Health, Stats... | Time, Day, Loc |
+------------------+--------------------------------------+
|                  |                                      |
|  [중앙 뷰포트]      |  [우측 사이드바 (Text-Only Actions)]    |
|                  |                                      |
|                  |  ACTIONS                             |
|  (배경+캐릭터 CG) |                                      |
|                  |  Standby     Action                  |
|                  |  Inventory                           |
|                  |                                      |
|                  |  ----------------------------------  |
|                  |  LUISE (Target-specific)             |
|                  |                                      |
|                  |  Talk        Touch                   |
|                  |  Gift                                |
|                  |                                      |
|                  |  ----------------------------------  |
|                  |                                      |
|                  |  Main Street South Gate              |
+------------------+--------------------------------------+

|  [루이제]: '어머, 오늘도 다치고 온 거예요...?'       |
+--------------------------------------------------------+

```

### 3.2. 세부 컴포넌트 스펙

#### A. 상단 표시줄 (Metric Bar)
* **방식:** 굵은 철제 프레임 내부에 완전한 텍스트 오버레이로 구현합니다.
* **좌측 나열:** 소지금(Funds), 체력(Health/HP), 정신력(Mental Health/SAN), 기타 스탯(Stats...). 미니멀한 텍스트로 가시성 확보.
* **우측 나열:** 현재 시각(Time), 요일(Day), 현재 위치(Location). 24시간제 및 노드 기반 장소 정보를 표시합니다.

#### B. 우측 사이드바 (Action Panel)
* **핵심 스펙:** 별도의 그래픽 버튼 이미지(TextureButton 등)를 사용하지 않습니다. 오직 **클릭 가능한 텍스트 레이블**로만 구성합니다.
* **구성:**
    * **타이틀:** 'ACTIONS' (큰 폰트)
    * **일반 행동:** Standby, Action, Inventory.
    * **대상 행동 (Target Actions):** 대상 NPC 이름(예: LUISE)을 헤더로 두고 하위에 행동(Talk, Touch, Gift)을 나열합니다. 장소와 조건에 따라 동적으로 추가/삭제되며, Ink 엔진의 `choice` 데이터를 직접 텍스트로 변환하여 출력합니다.
    * **이동 (Movement):** 하단에 장소 이름(예: Main Street, South Gate)을 나열합니다. 'MOVEMENT' 타이틀을 삭제하여 더 많은 장소 목록을 텍스트로 촘촘하게 배치합니다.

#### C. 하단 대화창 (Dialogue Bar)
* **방식:** 오래된 가죽 질감의 배경 영역에 텍스트만 출력합니다.
* **구성:** `[NPC이름]: "대사 내용"` 형식 유지. VN 인터페이스의 메시지 창 역할을 하며, Ink 내러티브 데이터를 직접 출력합니다.

## 4. UI 상호작용 및 시각 피드백 (v1.2 수정)

고전 텍스트 어드벤처 느낌을 살리기 위해, 텍스트 자체의 상태 변화를 피드백으로 사용합니다.

### 4.1. 텍스트 상태 피드백 (Hover/Selected)
* **Normal:** 옅은 아이보리색 (#EAE6DF).
* **Hover (마우스 오버):** 크림슨 레드 (#A91D22) 색상 변경 및 밑줄(Underline) 활성화.
* **Selected (클릭):** 텍스트가 살짝 확대되며 더 밝은 빨간색으로 점멸한 뒤 행동 실행.
* **결정 사유:** 별도의 버튼 애니메이션 없이 텍스트 자체의 변화만으로 스타일리시한 고전 감성을 전달하기 위함.

## 5. 구현 우선순위 및 로드맵

1. **Phase 1: 기능 구현 (v1.2 스펙):** * Godot에서 `Label` 노드를 활용해 `image_23.png`와 동일한 텍스트 기반 UI 레이아웃 구현.
    * Ink 엔진과 연동하여 스탯 변화 및 ACTIONS 목록이 동적으로 텍스트로 출력되는 시스템 완성.
2. **Phase 2: 비주얼 폴리싱 (추후 진행):** * 굵은 철제/가죽 프레임 테두리의 텍스처를 고해상도로 교체. 폰트 세부 조정.
    * 텍스트 팝 효과(위기 시 폰트 흔들림 등) 애니메이션 추가.

이 스펙 문서를 토대로 1단계 기능 구현에 집중하며, 텍스트 어드벤처 특유의 서사적 몰입도를 빠르게 검증할 수 있을 것입니다.
