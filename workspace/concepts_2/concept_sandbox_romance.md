# 컨셉: 모험가 마을 샌드박스 연애 시뮬

## 핵심 컨셉
**자유로운 샌드박스 판타지 연애 게임**. 플레이어는 모험가가 되어 한 마을에서 자유롭게 생활하고, 수많은 NPC들과 관계를 쌓으며 연애를 즐긴다. 삼각관계, 질투, 우정-연애의 경계 등이 자연스럽게 발생하는 것이 핵심 차별점.

## 세계관
암흑 에너지의 폭발이 전 지역을 뒤덮고 시공간이 뒤틀린 상황. **아울네스트**라는 위험지대의 마을에서 주인공은 모험가로서 활동한다. 주인공은 마을의 여관을 거점으로 하며, 다른 모험가들, 상인들, 주점 단골들과 어울리며 삶을 꾸려간다.

## 핵심 차별점: 삼각관계 중심 연애 시스템

### 기존 연애 게임 vs 이 게임
| 기존 연애 게임 | 이 게임 (샌드박스) |
|---------------|------------------|
| 히로인 A, B, C 중 하나를 선택하여 루트 진행 | 모든 NPC와 동시에 관계를 맺음 |
| 선택지로 특정 히로인 호감도 상승 | 행동(시간 배분, 대화 선택)으로 관계 변화 |
| 루트 진행 중 다른 히로인과는 거의 교류 없음 | A와 친해지면 B가 질투함 |
| 엔딩 후 재플레이 | 엔딩 없음. 계속된 삶의 반복 |

### 삼각관계 발생 메카닉
```
예시 시나리오:
- 루이제와 매일 저녁 주점에서 대화 → 호감도 상승
- 동시에 미아(여 마법사)와 던전 파티를 자주 함 → 호감도 상승
- 둘 다 주점에 있을 때 → 자연스럽게 삼각관계 이벤트 발생
- 둘 중 한 명을 선택해서 대화하면 → 다른 한 명의 호감도 감소 + 질투 이벤트
```

### 관계 네트워크 시스템
- 모든 NPC는 서로 **독립적인 관계**를 가짐
- A NPC가 B NPC와 친하면, 플레이어가 A와 친해지면 B와의 관계도 간접 상승
- 반대로 A와 적대관계인 C가 있으면, A와 너무 친해지면 C와의 관계는 하락
- **커뮤니티 분위기**가 형성됨: 특정 NPC들끼리만 모이는 무리가 생기고, 플레이어는 그중 하나에 가담하거나 중립을 지킴

## 캐릭터 구성

### NPC 구조 (예시: 12~16명)
```
[주점/여관 그룹]
- 루이제: 주점 주인 (원작 NPC)
- 린다: 웨이트리스, 루이제와 자매처럼 지냄
- 브란든: 노총각 바텐더, 말수 적음

[모험가 그룹]
- 미아: 마법사, 루이제와 경쟁의식 (히든 삼각)
- 그렉: 전사, 미아를 좋아하지만 주인공과 친함 (브로콜리 삼각)
- 실비아: 도적, 독립적, 주인공에게 호감 있음

[상인/마을 주민 그룹]
- 마르코: 무기상, 루이제 옛날 알바 동료
- 엘레나: 포션상, 그렉과 교제 중 (주인공이 끼어들 가능)
- 올리버: 마을 경비대장, 엄격하지만 술 취하면 헌신적

[히든/특수 캐릭터]
- ???: 특정 조건 만족 시 등장하는 비밀 NPC
```

### 각 NPC 특성
- **성격 태그**: 금사빠, 츤데레, 순진함, 질투심 많음, 독립적, etc.
- **취향**: 어떤 선물을 좋아하는지, 어떤 대화 주제에 반응하는지
- **스케줄**: 특정 시간에 어디에 있는지 (주점/던전/상점/집)
- **관계망**: 누구와 친하고, 누구와 사이가 나쁜지

## 게임플레이 시스템

### 1. 시간 진행 시스템
```
아침 (06:00) → 점심 (12:00) → 저녁 (18:00) → 밤 (22:00) → 새벽 (02:00)

- 각 시간대마다 NPC들은 고유한 위치/행동 패턴
- 플레이어는 "이 시간에 누구를 만날까"를 전략적으로 선택
- 여러 NPC가 같은 장소에 모이면 집단 이벤트 발생 가능성 ↑
```

### 2. 주점 대화 시스템 (핵심)
```
[주점 입장]
→ 현재 있는 NPC 목록 표시 (루이제, 미아, 그렉)
→ 행동 선택:
   - [루이제에게 말 걸기] → 1:1 대화
   - [미아에게 말 걸기]
   - [그렉에게 말 걸기]
   - [전체에게 한 마디] → 집단 대화 (삼각관계 트리거 가능)
   - [혼자 술 마시기] → NPC들이 먼저 말 걸어옴

[대화 중 이벤트 예시]
루이제: "오늘도 고생했어요?"
선택지:
- "루이제 씨는 항상 바쁘네요" → 루이제 호감 +, 대화 종료
- "미아 씨는 오늘 던전 갔다 왔나요?" (미아도 있을 때) → 삼각 시작
- "그렉이랑 둘이서..." (그렉과 둘만 있을 때) → 그렉과의 관계에 따라 질투/응원
```

### 3. 모험 시스템
```
던전은 단순한 텍스트 기반 선택지 모음

[던전 탐험 예시]
→ 던전 선택 (하수도 / 폐허의 탑 / 암흑 숲)
→ 파티 구성 (혼자 / 미아와 / 그렉과 / 새로운 NPC와)
→ 이벤트 진행:
   HP: 80/100 | 정신: 15/20 | 탐색도: 3/10
   "고블린 무리가 보인다"
   [돌진한다] [회피한다] [대화 시도]
→ 결과에 따라 보상/전투/이벤트 발생
→ 귀환 시각 결정 (저녁 늦게 귀환 → 루이제 걱정 이벤트)
```

### 4. 호감도 & 관계 시스템
```
호감도 스펙트럼: -100 (증오) ~ +100 (사랑)

단계별 변화:
- -100 ~ -50: 적대
- -50 ~ 0: 무관심
- 0 ~ +20: 흥미
- +20 ~ +50: 친구
- +50 ~ +80: 연인 후보 (삼각관계 본격화)
- +80 ~ +100: 연인 (단, 다른 사람과도 +80 이상이면修羅場)

관계 특수 상태:
- [질투]: 같은 장소에서 다른 NPC와 친밀한 행동 시 발생
- [의심]: 둘 다 높은 호감을 가진 NPC들이 만났을 때
- [우정]: 높은 호감이지만 연애로 발전하지 않는 관계
- [숨김]: 비밀 연애 (다른 NPC들에게 발각되면 폭풍)
```

### 5. 삼각관계 이벤트 시스템
```
발생 조건:
- 두 NPC A, B 모두 플레이어에게 +40 이상 호감
- A와 B가 같은 장소에 있음
- 플레이어가 둘 중 한 명에게 호감 표현 행동

이벤트 타입:
1. 수동적 관찰: A가 플레이어와 B의 대화를 지켜봄 → A의 행동/표정 변화
2. 직접 개입: A가 끼어들어 "우리도 얘기하자" → 선택지 발생
3. 뒷이야기: B가 없는 자리에서 A가 "그 사람 좋아하는 거야?"라고 물음
4. 정면충돌: A와 B가 동시에 데이트 신청 → 선택 강요
5. 정보 유출: C가 A에게 "플레이어가 B랑 같이 던전 갔대"라고 전달

선택지 결과:
- 한 쪽 선택 → 선택받지 못한 쪽 호감도 대폭 하락 + 질투 플래그
- 둘 다 거절 → 둘 다 실망하지만 질투는 해소
- 회피 → 둘 다 의심 플래그
```

### 6. 이벤트 가시성 분기
```
핵심 원칙: "본 것 / 못 본 것"으로 스토리가 갈림

예시 1:
- 미아의 생일 이벤트를 본 플레이어 → 미아에게 특별한 대화 선택지 추가
- 못 본 플레이어 → 평소와 같은 대화

예시 2 (삼각):
- 루이제가 그렉에게 차이는 장면을 봄 → 루이제 위로/기회주의 선택
- 못 봄 → 루이제의 행동 변화를 이해하지 못함

결과: 플레이어마다 완전히 다른 "연애 스토리"가 생성됨
엔딩이 아닌 "현재 관계 상태"가 플레이어의 경험을 결정
```

## 게임 진행 구조

### 샌드박스 루프
```
1일 사이클:
[아침] 준비 → [던전/마을 활동] → [저녁 주점] → [대상 선택/관계 진행] → [밤] → 다음날

무한 반복. 특정 목표 없음.
플레이어가 원할 때:
- 새로운 관계 도전
- 현재 관계 유지
- 모든 관계 정리하고 새로 시작 (리셋 옵션)
```

### 리플레이 밸류
- 첫 플레이: 루이제에게 집중 → 루이제 루트 체험
- 두 번째: 미아와 그렉 사이에서 삼각관계 체험
- 세 번째: 브란든(바텐더) 히든 루트 발견
- 네 번째: 엘레나와 그렉 커플을 깨는 악역 플레이
- 다섯 번째: 모두와 친구로 지내는 수호천사 플레이

## 아트 리소스 전략

### 배경 에셋 최소화
- 주점 내부 (1장)
- 던전 텍스트 묘사 + 간단한 분위기 이미지 (3~4장)
- 마을 거리 (1장)
- NPC 방/집 (중요 NPC만 2~3장)

### 캐릭터 에셋 집중
- 각 NPC: 스탠딩 (표정 3~5개) + SD아이콘
- 의상 변화: 없음 (단순화)
- CG: 삼각관계 이벤트용 (3~5장만)

## 기술적 구현

### 핵심 필요 기능
1. 시간 진행 시스템
2. NPC 스케줄/위치 관리
3. 호감도 계산 (복수 NPC 간 상호작용 포함)
4. 삼각관계 이벤트 트리거
5. 가시성 분기 관리 (플레이어마다 다른 이벤트 플래그)
6. 대화 시스템 (개인/집단)

### 난이도 평가
- 단일 히로인보다 복잡함 (관계 매트릭스 필요)
- 하지만 전투/던전이 텍스트 기반이라 프로그래밍은 제한적
- 주요 작업량: 시나리오 (대사량 폭증)

## 차별화 요약

| 요소 | 차별점 |
|-----|--------|
| **자유도** | 시간대/장소/대상 완전 자유 선택 |
| **삼각관계** | 자연 발생, 질투 시스템, 정보 유출 |
| **관계망** | NPC들끼리도 관계가 있어 플레이어 선택이 파급력 있음 |
| **엔딩 없음** | 샌드박스. 플레이어가 만드는 스토리가 곧 게임 |
| **리플레이** | 매 플레이 다른 인간관계 네트워크 생성 |

## 참고 작품
- **eratouhou the world**: 다수 NPC, 호감도, 자유로운 상호작용
- **마비노기**: 마을 중심 생활, NPC와의 관계
- **하베스트 문/스토리 오브 시즌**: 마을 생활 + 연애
- **페르소나 시리즈**: 시간 관리 + 관계 시뮬레이션

## 예상 개발 기간
- 기간: 4~6개월 (단일 히로인 대비 2배)
- 주요 병목: 시나리오 (대사량 5~10배 증가)
- 추천 접근: 6~8명 NPC로 MVP 출시 → 업데이트로 추가
