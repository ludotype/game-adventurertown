# 컨셉: 벙커게임

## 핵심 컨셉
전체주의적 제국의 최전방 감시기지 벙커에서 근무하는 병사가 되어, 방어 시설 점검, 식사, 여가 활동을 하며 전선의 불온한 움직임을 감시하고 보고하는 게임. Papers Please / This War of Mine / Beholder 류의 스토리-헤비한 도덕적 선택 게임.

## 캐릭터
- **소꿉친구**: 제국의 프로파간다에 빠져 있고 주인공에게 맘이 있어 보이는 소꿉친구.
- **지역 사령관**: 엄하지만 사실은 자신의 임무에 회의감을 갖고 있는 지역 사령관.
- **전투용 메카**: 보급과 재충전을 위해 주인공의 벙커로 돌아오는, 평화를 동경하는 전투용 메카.

## 게임플레이
- 벙커 내부를 돌아다니며 일상 업무 수행.
- 라디오를 통한 통신 감청 및 보고.
- 피난민 구호 / 거부 / 포격 등의 도덕적 선택.
- 상부의 과도한 명령 거부 또는 따르기.
- 스토리 진행에 따른 주인공 및 NPC들의 운명 변화.

## 아트/스토리 방향성
사용자의 아트 디렉션과 유머가 섞인 어른스러운 스토리로, 어두운 전시 상황 속에서도 귀여운 미소녀 캐릭터들과의 인터랙션을 자연스럽게 녹여냄.

## 판매 전략
최대 10달러에 책정하여 빠르게 완성 후 판매. 부족하더라도 완성하여 판매하고, 개선하여 후속작을 내는 방식.

---

## 평가

### 1. 2개월 혼자 완성 가능성: B
- 단일 로케이션(벙커)이라 배경 리소스는 절약됨.
- 그러나 도덕적 딜레마가 효과적이려면 스토리 분기가 매우 많아야 하며, 시나리오 작업량이 기하급수적으로 늘어남.
- Papers Please는 국경 검문소 하나였지만, 그 안의 시스템 복잡도와 스토리 분기는 상당했음.
- 3명의 메인 캐릭터 각각과의 관계 분기 + 전시 상황 분기를 2개월 내에 혼자 작성하는 것은 빡빡함.

### 2. 참신함 / 인터랙티브함: A
- Papers Please를 명시적으로 벤치마킹한 구조라 참신함은 확실히 확보 가능함.
- 벙커라는 폐쇄 공간 + 전시라는 극한 상황 속에서의 일상과 선택은 긴장감 있음.

### 3. 귀여운 여자애들 / 재미있는 대사 적합성: B
- 사용자가 "뛰어난 아트 디렉션과 유머가 섞인 어른스러운 스토리"로 해결하겠다고 했으나, 전쟁/전시라는 무거운 주제와 귀여운 아트/유머의 조화는 여전히 까다로움.
- 3명의 캐릭터(소꿉친구, 사령관, 메카)는 설정상 흥미롭고 아트로 차별화할 여지가 큼.
- 단, "유머가 섞인 어른스러운 스토리"를 2개월 내에 Papers Please 수준으로 구현하는 것은 시나리오 작업량이 엄청남.

### 4. 비주얼 노벨 엔진 연계성: B
- 2D VN 엔진으로 구현 자체는 가능함.
- Papers Please의 핵심은 문서 검사, 도장, 승인/거부 등의 UI 인터랙션인데, 이를 VN 엔진에서 자연스럽게 재현하려면 어느 정도의 커스텀 UI 작업이 필요함.
- 라디오 통신 시스템, 감시 장비 UI 등도 추가 프로그래밍 필요.

### 5. 최소 프로그래밍 필요도: 중간~높음
- 도덕적 선택 분기, 엔딩 분기, 일상 루틴 시스템, 감시 UI 등 프로그래밍이 필요함.

### 6. $10 판매 적합성: B
- 단편 스토리 게임으로서 $10 이하 가격대는 적합함.
- 그러나 2개월 내 완성품의 콘텐츠량이 $10 가치를 줄 수 있을지는 미지수. Papers Please는 1인 개발로 9개월 이상 걸림.

### 종합
참신함은 있으나, Papers Please급 서사를 2개월 내에 혼자 구현하는 것은 현실적으로 매우 어려움. 사용자의 아트 강점을 살릴 수는 있으나 시나리오 복잡도가 과도함. **보류**.
