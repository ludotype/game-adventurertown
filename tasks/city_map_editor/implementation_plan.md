# 🗺️ Task: City Map Editor (GraphEdit & Inspector Sidepanel Redesign)

본 문서는 에디터의 투박한 조작성과 데이터 동기화 문제를 완벽하게 타개하기 위해, **'노드 비주얼 슬림화'** 및 **'우측 속성 검사기(Inspector) 사이드바 패널'**을 전면 도입하는 리팩토링 설계 명세서입니다.

---

## 1. 현재 구현의 문제점 및 리팩토링 배경

Kimi 에이전트의 1차 구현 결과, 플러그인의 뼈대는 완성되었으나 다음과 같은 치명적인 한계점이 발견되었습니다:

1. **[데이터 무결성 결함] 연결 데이터의 ID 불일치**: 
   * `connections.json`에 Godot의 런타임 임시 노드명(`@@35` 등)이 그대로 저장되어 게임 엔진이 맵 구조를 읽지 못하거나, 재로딩 시 선 연결 복구가 불가능한 현상.
2. **[UI 조작 편의성 결함] 장소 노드 내부의 심각한 과밀화**:
   * 노드 내부에 입력칸 3개와 이벤트 버튼들이 무식하게 밀어 넣어지면서, 노드가 시각적으로 빽빽해지고 텍스트가 잘리는 등 투박한 UX를 낳음.
3. **[이벤트 뷰 분리 유실]**: 
   * 월드 맵의 노드가 너무 비대해져서 줌아웃 시 맵 전체의 연결망을 조망하기 힘들어짐.

---

## 2. 해결책 및 아키텍처 리디자인 (Refined Architecture)

이 부족함을 해결하기 위해 마스터마인드로서 다음과 같이 개선 설계를 확정합니다.

```
+--------------------------------------------------------------------------------+
|  [Save] [Load]                                                                 |
+-------------------------------------------------------+------------------------+
|                                                       | [장소 상세 속성 패널]   |
|   (North District)                                    |                        |
|                                                       | 장소 ID:               |
|   +-------------------+       +-------------------+   | [ fireside_amber     ] |
|   |  루이제의 여관    |======>|   골동품 상점     |   |                        |
|   | (fireside_amber)  |       |   (curio_shop)    |   | 장소 한글명:           |
|   +-------------------+       +-------------------+   | [ 루이제의 여관       ] |
|                                                       |                        |
|                                                       | 상주 NPC (쉼표 구분):  |
|                                                       | [ luise, inn_staff   ] |
|                                                       |                        |
|                                                       | +--------------------+ |
|                                                       | | + 이벤트 추가      | |
|                                                       | +--------------------+ |
|                                                       | - Ev: 누군가 탄 약     |
|                                                       |   [대화열기] [삭제]    |
|                                                       |                        |
+-------------------------------------------------------+------------------------+
```

### 2.1 장소 노드(`map_place_node.gd`)의 슬림화 (Slim Nodes)
* **목표**: 노드 내부에 텍스트 입력창(`LineEdit`)이나 버튼을 **단 하나도 두지 않고**, 오직 시각적 정보 표시에만 집중합니다.
* **구성**:
  * 타이틀: `display_name_kr` (한글명)
  * 라벨 1: `place_id` (장소 고유 ID)
  * 런타임 연결 포트만 좌우에 깔끔하게 배치.
* **노드명 동기화**: 노드 인스턴스 생성 또는 데이터 로드 시, Godot 노드의 `name` 속성을 장소의 `id`와 완전 일치시킵니다 (`node.name = data.id`). 
  * 이렇게 하면 `connections.json`에 `@@35` 같은 임시 이름 대신 실제 장소 ID(`fireside_amber` 등)가 저장되므로 데이터 바인딩 문제가 우아하게 해결됩니다.

### 2.2 우측 상세 속성 패널 (Detail Sidebar Panel) 신설
* `map_editor_dock.tscn`을 가로 분할 컨트롤(`HSplitContainer`) 구조로 레이아웃을 개편합니다.
  * **좌측 영역 (75% 비율)**: 광활하고 깔끔해진 `GraphEdit` 캔버스 영역.
  * **우측 영역 (25% 비율)**: 선택된 노드의 디테일한 속성과 이벤트 목록을 조작할 수 있는 **사이드바 컨테이너(VBoxContainer + ScrollContainer)** 영역.
* **상호작용 플로우**:
  1. `GraphEdit`에서 특정 노드를 마우스 왼쪽 클릭하여 선택합니다 (`node_selected` 시그널 격발).
  2. 우측 상세 패널이 활성화되며, 선택된 노드 객체(Node Reference)를 캐싱합니다.
  3. 상세 패널 내의 입력창에서 이름이나 ID를 수정하면, 실시간으로 선택된 노드의 텍스트와 타이틀이 동적으로 동기화됩니다.
  4. 이벤트의 추가/삭제/대화 열기 조작은 오직 이 우측 패널에서만 쾌적하게 수행합니다.

## 3. 리팩토링을 반영한 신규 데이터 스키마 규칙

### 3.1 연결선 저장 구조 정화
* `connections.json`에는 오직 장소 ID 간의 전이 데이터만 기록합니다:
```json
{
  "connections": [
    {
      "from": "fireside_amber",
      "to": "curio_shop",
      "movement_cost": 1
    }
  ]
}
```

---

## 4. 최종 인수 기준 (Acceptance Criteria)

* [ ] 월드 맵의 노드에는 입력창이나 버튼이 없으며, 오직 장소 이름과 ID만 표기됨.
* [ ] 노드를 클릭하면 우측 상세 패널이 매끄럽게 활성화되고, 빈 곳을 클릭하면 상세 패널이 비워짐.
* [ ] 상세 패널에서 한글 표시명을 타이핑해 변경하면 실시간으로 해당 그래프 노드의 타이틀이 업데이트됨.
* [ ] 저장하고 다시 불러왔을 때, `connections.json`에 `@@` 로 시작하는 Godot 임시 노드명 대신 실제 장소 ID가 완벽히 저장 및 로드 복구됨.
* [ ] 이벤트의 추가, 조건(Trigger) 작성, dialogue 파일 경로 매핑 및 Dialogue Manager 편집기 탭으로 점프 연동이 우측 상세 패널 내에서 온전히 정상 작동함.
