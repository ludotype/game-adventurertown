# 💬 대화 이벤트 맵 에디터 UX 고도화 및 파일/라벨 자동 생성 구현 계획서

사용자님이 리포트와 함께 제안해주신 피드백을 기반으로, 맵 에디터의 실전 기획 편의성을 대폭 극대화하는 UX 고도화 및 자동화 설계를 제안합니다.

---

## 1. 주요 변경 및 개선 계획

### 1.1 캔버스 노드 카드 시각화 슬림화
* **기존 문제**: 노드 내부에 매핑된 이벤트의 개별 이름과 점프 단추가 모두 나열되어, 장소에 수십 개의 대화가 엮일 시 노드의 크기가 비정상적으로 커지고 화면이 극도로 복잡해집니다.
* **해결 방안**: 캔버스 상의 개별 노드 본체에는 이벤트 이름들을 일일이 노출하지 않고, 상주 NPC 표시처럼 **`💬 매핑된 대화 이벤트: X개`** 형태의 콤팩트한 통계 라벨 하나만 연보라색으로 출력하여 수직 블로트를 완벽히 차단합니다.

### 1.2 대화 파일 선택창(📁 탐색기) 도입
* **기존 문제**: 대화 스크립트 리소스 경로(`res://data/dialogues/test.dialogue`)를 키보드로 일일이 기재해야 하므로 오타에 취약하고 불편합니다.
* **해결 방안**: 이벤트 팝업 내 경로 입력창 우측에 **폴더 아이콘 버튼(📁)**을 신설하고, 클릭 시 Godot의 네이티브 **`EditorFileDialog`**를 호출하여 프로젝트 내의 `.dialogue` 파일을 마우스 클릭만으로 쉽게 선택할 수 있게 개선합니다.

### 1.3 대화 파일 및 타이틀 라벨 자동 생성/연동 (가장 중요 🌟)
* **기존 문제**: `🔗 점프`를 눌렀을 때 파일이 없거나 라벨이 다르면 에러가 나거나 수동으로 파일을 만들어야 합니다.
* **해결 방안**: `_jump_to_dialogue` 동작 시 다음 자동화 규칙을 주입합니다.
  1. **파일 미존재 시**: 해당 경로에 디렉토리를 자동 생성하고, 지정한 라벨이 선언된 빈 `.dialogue` 템플릿 파일을 새로 만듭니다 (`~ [라벨명]` 및 안내 주석 포함).
  2. **파일 존재 + 라벨 미존재 시**: 기존 `.dialogue` 파일을 읽어 지정한 라벨 선언(`~ [라벨명]`)이 없을 경우, 파일 끝에 새 대화 섹션을 자동으로 덧붙여(Append) 줍니다.
  3. **최종 동작**: 생성/보완된 대화 파일의 라벨 위치로 `DMPlugin.open_file_at_title`을 사용하여 완벽하게 정밀 점프합니다.

## 3. 검증 계획 (Verification Plan)

### 수동 검증 단계 (Manual Verification)
1. Godot 에디터를 기동(또는 플러그인 리로드)하여 `City Map Editor`를 활성화합니다.
2. 특정 장소 노드(예: `easttown_street_1`)를 선택하고 사이드바에서 `💬 대화 이벤트 편집기 열기`를 누릅니다.
3. 팝업창 내부에서 `+ 새 스토리 대화 이벤트 추가`를 클릭합니다.
4. 새로 기입할 경로 필드 우측의 `📁` 탐색기 버튼을 클릭하여 기존의 대화 파일을 선택해 봅니다.
5. 존재하지 않는 경로(예: `res://data/dialogues/new_quest.dialogue`)와 라벨(예: `meet_luise`)을 적어봅니다.
6. `🔗 점프`를 누른 뒤:
   * 실제로 `res://data/dialogues/new_quest.dialogue` 파일이 신설되었는지 확인합니다.
   * 해당 파일 안에 `~ meet_luise` 라벨이 올바르게 선언되었고 Dialogue Manager 에디터 창이 그 라벨을 지목하고 있는지 검수합니다.
7. 기존 대화 파일을 대상으로 존재하지 않는 라벨명(예: `luise_secret_story`)을 적고 `🔗 점프`를 눌렀을 때, 기존 파일 뒷부분에 해당 라벨 블록이 잘 이어 붙어 생성되는지 검수합니다.
8. 다이얼로그를 닫았을 때, 캔버스 노드 카드가 지저분해지지 않고 오직 `💬 매핑된 대화 이벤트: 2개`로만 산뜻하게 표시되는지 검증합니다.
