## 1. 평가 이력

### 1.1 1차 평가: 길드 관리 시뮬레이션 (2026-05-16, wiki 정리 전)

**기획 내용:**
- 장르: 관리형 전략 시뮬레이션 + 자동 전투 RPG
- 플레이어는 길드 마스터가 되어 모험가를 고용/관리
- 자동 전투, 프로시저럴 맵, 건설 시스템, 평판 시스템
- 참고작: Loop Hero, Idle Heroes, Darkest Dungeon

**평가 결과:**
- **제작 용이성 3/10**: 1인 개발에 비해 스코프가 과도. 3-4개월 MVP는 비현실적.
- **컨셉 참신함 4/10**: "자동 전투 + 관리"는 이미 모바일 오터게임에서 흔한 조합. PC Steam 유저에게 어필하기 어려움.
- **스팀 경쟁력 5/10**: "자동 전투"는 PC 게이머에게 모바일 연상. $4.99 가격대와 10-30시간 플레이타임은 콘텐츠 부족으로 인식될 수 있음.

**핵심 문제:** 기획서와 실제 코드가 충돌. `Game_Concept.md`는 관리 시뮬을 기술했지만, `architecture.md`와 실제 코드베이스는 프린세스 메이커/스타듀 밸리형 라이프 시뮬을 구축하고 있었음. 설계 정체성의 이중화.

### 1.2 2차 평가: 1인칭 연애 어드벤처 (2026-05-16, wiki 정리 후)

**기획 내용:**
- 장르: 1인칭 판타지 연애 어드벤처 + 텍스트 어드벤처 북
- 플레이어는 위험지대 마을의 모험가. 여관 주인 루이제와의 연애가 핵심.
- 던전: 텍스트 기반 선택지 분기. 전투 최소화.
- 엔딩 없음. 상태 유지.
- 참고작: eratouhou the world, 마비노기, 하베스트 문, 페르소나

**평가 결과:**
- **제작 용이성 8/10**: 마을/연애 엔진이 이미 70-80% 완성. 텍스트 어드벤처는 별도 전투 시스템 없이 VN 인터페이스로 구현 가능.
- **컨셉 참신함 6/10**: 단일 히로인은 집중의 장점이지만 Steam VN 시장에서 다수 캐릭터가 기본. "엔딩 없음"은 완성감 부족 우려.
- **스팀 경쟁력 7/10**: $10 가격대 적정. PPB 스핀오프 마케팅 각 유효. 아트 퀄리티가 높다면 위시리스트 전환율 강력.

**핵심 문제:**
- 던전과 연애의 연결고리 약함 (늦은 귀환 → 걱정 이벤트만)
- 텍스트 어드벤처 북은 스트리밍/관전성 약함
- 단일 히로인 = 팬덤 형성 어려움

### 1.3 3차 평가: 1인칭 연애 어드벤처 + 루프 히어로식 전투 (2026-05-16, 추가 정보 반영)

**추가된 정보:**
- 아트: 스탠딩 CG 1장 + 표정 10종 = 이틀. 매우 빠른 파이프라인.
- 히로인: 루이제 MVP → 2명 추가 예정 (총 3명). 데이터 드리븐 설계로 확장성 확보.
- 가격: $15 이하 (히로인 추가 시 $10-14 예상)
- 던전: 루프 히어로식 사이드 스크롤 자동 전투 + Eldritch Horror 장비/상태이상/스킬 시스템
- 몬스터: PPB 프로젝트에서 사용하던 고퀄리티 도트 스프라이트 수십 종 이미 구현됨
- Localization: EN/CN/KO/JA 발매일 지원. 번역가 출신이 직접 처리.
- 엔딩: 캐릭터당 다중 엔딩 + 게임오버 이벤트 (루이제 납치 등)
- 모드: Endless Mode / Story Mode 기본 + Ironman Mode (로그라이크식 풀 리셋) 별도
- 시점: 1인칭 → 사이드 스크롤로 변경 (PPB 에셋 활용)

**평가 결과:**
- **제작 용이성 7/10**: 마을 엔진 완성 + 빠른 아트 파이프라인 + AI 보조 시나리오 + 자체 번역. 다만 전투 시스템 구현(2-3주)과 EH 시스템 접목(1-2주)이 추가 부담.
- **컨셉 참신함 8/10**: 사이드 스크롤 자동 전투 + VN 연애 + EH 장비/상태이상의 조합은 시장에서 거의 볼 수 없는 하이브리드. 3 히로인 + 다중 엔딩으로 리플레이 밸류 확보.
- **스팀 경쟁력 8/10**: $10-15 가격대에 4개국어 발매일 지원은 엄청난 강점. 특히 중국어 시장(Steam VN/연애 시뮬 수요 폭발적). 사이드 스크롤 전투는 스트리밍에 유리. Ironman 모드는 로그라이크 팬층 확보.

## 2. 상세 평가

### 2.1 제작의 용이성

**강점:**

1. **엔진이 이미 완성되어 있다.** PlaceRegistry, NPCRegistry, InteractionRegistry, TimeSystem, MetricStore, ActionRunner, ConditionEvaluator, DialogueManager 연동까지 전부 구동 중. 이건 2개월+의 개발 시간을 절약한 것이다.

2. **아트 파이프라인이 매우 빠르다.** 스탠딩 CG + 표정 10종 = 2일. 3 히로인 = 6일. 이는 업계 기준으로도 매우 빠른 속도이며, "콘텐츠 작업량이 병목"이라는 일반적인 인디 개발의 저주를 상당 부분 해소한다.

3. **시나리오 작업도 빠르다.** AI 보조 + 본인의 작문 강점. 데이터 드리븐 아키텍처(JSON 기반) 덕분에 대사 추가는 프로그래머 개입 없이 기획자/작가가 직접 처리 가능.

4. **번역이 전문가 수준으로 자체 처리 가능.** EN/CN/KO/JA 4개국어를 발매일부터 지원하는 것은 인디 게임에서 거의 볼 수 없는 경쟁력. 특히 중국어 시장은 VN/연애 시뮬 소비가 매우 높다.

5. **PPB 에셋 재활용.** 수십 종의 몬스터 스프라이트가 이미 구현되어 있어, 던전 전투 파트의 아트 작업량이 거의 0이다.

**약점:**

1. **전투 시스템 구현이 필요하다.** 루프 히어로식 사이드 스크롤 자동 전투는 구현이 단순하지 않다:
   - 몬스터 스프라이트 표시 시스템
   - 자동 전투 해상 로직 (스탯 비교, 데미지 계산)
   - HP/MP바, 데미지 넘버 표시
   - 스피드 컨트롤 (1x/2x/4x)
   - 장비 장착/해제 UI
   - 상태이상 표시 및 턴 처리
   예상: 2-3주

2. **Eldritch Horror 시스템 접목.** 다이스 풀, 상태이상 카드, 레코닝(reckoning) 시스템은 기획 문서(workspace/systems/)에 이미 있지만, Godot에서의 구현은 시간이 소요된다. 예상: 1-2주.

3. **3 히로인분의 콘텐츠.** 1 히로인에서 3 히로인으로 확장하면 interaction JSON, 대화 파일, 이벤트, 엔딩 분기가 3배. 각 히로인당 예상 2-3주의 시나리오 작업.

4. **세이브 시스템이 깨져 있다.** SaveManager가 구 컨셉(GameManager, Flags.sanity 등)을 참조 중. 현재 아키텍처와 호환되도록 수정 필요.

5. **인벤토리 UI가 껍데기뿐.** `inventory_window.tscn`은 존재하지만 스크립트(`inventory_window_logic.gd`)가 없다. 장비/선물/전리품 시스템에 필수.

**예상 타임라인:**

| 항목 | 소요 |
|------|------|
| 전투 시스템 구현 | 2-3주 |
| 장비/상태이상 시스템 | 1-2주 |
| 루이제 컨텐츠 (대사/이벤트/엔딩) | 2-3주 |
| 추가 2 히로인 | 각 2-3주 |
| 전투 밸런싱 | 1-2주 |
| 4개국어 번역 | 2-3주 (AI 보조) |
| 폴리시/디버그 | 2-3주 |
| **총 MVP (루이제 단독)** | **약 3개월** |
| **총 3 히로인 + 전투** | **약 5-6개월** |

**결론:** 루이제 단독 MVP → 얼리 액세스 → 히로인 추가 → 1.0 출시의 단계적 로드맵이 현실적이다.

### 2.2 컨셉의 참신함

**강점:**

1. **시장에서 거의 본 적 없는 하이브리드.** VN 연애 시뮬 + 사이드 스크롤 자동 전투 RPG + Eldritch Horror 장비/상태이상 시스템의 3종 결합은 독특하다. 비슷한 조합을 찾기 어렵다.

2. **"전투의 결과가 관계의 운명을 바꾼다."** 던전에서 얻은 전리품/의뢰 완수/장비가 루이제와의 관계에 직접 영향을 준다는 연결고리는 기존 연애 게임에서 볼 수 없는 구조.

3. **사이드 스크롤 자동 전투의 1인칭 서사 결합.** 루프 히어로는 탑뷰 루프지만, 이 게임은 "내 캐릭터가 마주 보는 몬스터와 싸운다"는 1인칭적 긴장감 + 사이드 스크롤의 시각적 피드백을 동시에 가진다.

4. **Ironman 모드.** 로그라이크식 풀 리셋 모드는 완전히 다른 플레이어층(도전 과제 헌터, 로그라이크 팬)을 끌어들인다. 같은 게임을 다른 방식으로 즐기는 메커니즘.

5. **게임오버 이벤트(루이제 납치).** 엔딩 없음의 단점을 보완하면서도, 스토리적 긴장감과 스테이크를 부여. "오늘 밤 루이제가 안전할까"가 플레이어의 무의식적 배경이 된다.

**약점:**

1. **PPB 스핀오프라는 점이 IP 제약이 될 수 있다.** 루이제를 모르는 신규 플레이어에게는 "이 게임의 세계관이 뭔가?"라는 질문이 생길 수 있다. 세계관 설명이 튜토리얼에서 명확히 되어야 한다.

2. **자동 전투에 대한 인식.** "자동 전투 = 모바일 게임"이라는 PC 게이머의 선입견이 여전히 존재. 루프 히어로나 슬레이 더 스파이어의 성공이 이를 완화하지만, 마케팅에서 "전략적 준비의 결과를 지켜본다"는 각도로 포지셔닝해야 한다.

3. **Eldritch Horror 시스템의 학습 곡선.** 다이스 풀, 상태이상 카드, 레코닝 등은 보드게임꾼에게는 익숙하지만 캐주얼 플레이어에게는 복잡할 수 있다. 튜토리얼과 점진적 도입이 필수.

### 2.3 스팀 경쟁력

**강점:**

1. **아트 퀄리티가 VN/연애 시뮬의 가장 중요한 구매 결정 요인이다.** Steam에서 위시리스트 → 구매 전환율은 스크린샷의 캐릭터 CG 퀄리티가 좌우한다. 루이제의 스탠딩 + 표정 변화가 뛰어나면 이것만으로도 팔린다.

2. **4개국어 발매일 지원은 엄청난 강점.** 특히:
   - **중국어**: Steam 중국 시장은 VN/연애 시뮬 소비가 폭발적. 미소녀 게임, 하베스트 문류가 중국어 지원 하나로 매출 3-5배 뛰는 사례가 많다.
   - **일본어**: VN 본고장. 일본 Steam 유저의 VN 수요가 꾸준하다.
   - **영어**: Steam 메인 마켓. 발매일 지원이 없으면 리뷰/매출이 크게 제한된다.
   - 번역가 출신의 직접 작업이라 품질 차이가 리뷰에 직결된다.

3. **$10-15 가격대는 적정 구간.** 비교:
   - Doki Doki Literature Club Plus: $15
   - Stardew Valley: $15
   - Loop Hero: $15
   - Coral Island: $25
   3 히로인, 다중 엔딩, 4개국어, 전투 시스템이 있으면 $15의 정당성이 충분하다.

4. **다중 히로인 + 다중 엔딩 = 리플레이 밸류.** 3 히로인 × 다중 엔딩은 "100% 완료"를 목표로 하는 Steam 플레이어에게 강한 동기가 된다. 이는 가치 제안이 "$15에 수십 시간"이 되게 한다.

5. **스트리머/콘텐츠 크리에이터 친화적.** 사이드 스크롤 자동 전투는 몬스터 스프라이트가 보이고 피해 숫자가 뜨므로, 텍스트 묘사만으로 전투를 진행하는 것보다 훨씬 관전성이 높다. "이 몬스터 봐봐, 귀여운 쥐가 검 든 거 봐" 같은 바이럴 가능.

6. **Ironman 모드는 니치 마케팅 채널이다.** 로그라이크 커뮤니티, 도전 과제 헌터들이 관심을 가질 수 있다.

**약점:**

1. **"GuildMaster"라는 타이틀이 현재 컨셉과 맞지 않는다.** 게임은 길드 관리가 아닌 모험가의 연애를 다룬다. 타이틀 변경을 검토해야 한다. (예: "Adventure Town", "Owl's Nest", "Hotel Morgana" 등)

2. **VN + 사이드 스크롤 전투의 전환 매끄러움.** 마을 장면(VN 인터페이스)에서 던전(사이드 스크롤 전투)으로의 전환이 어색하지 않아야 한다. 페이드/전환 효과와 일관된 UI 테마로 해결 가능.

3. **PPB IP에 대한 신규 플레이어의 진입 장벽.** 루이제를 모르는 유저에게 세계관이 직관적으로 전달되어야 한다. 인트로 시퀀스와 튜토리얼에서 "암흑 에너지 폭발 → 위험지대 마을 → 당신은 모험가"를 30초 안에 설명해야 한다.

## 4. 남은 리스크 및 해결 방안

### 4.1 전투-연애 연결고리 강화 (기획 리스크)

**문제:** 던전 전투의 결과가 루이제와의 관계에 어떻게 영향을 주는가? 현재 "늦은 귀환 → 걱정 이벤트"만으로는 충분하지 않다.

**해결 방안:**
- **선물 시스템**: 던전에서 희귀 재료/꽃을 얻어 루이제에게 전달
- **의뢰 완수**: 루이제가 부탁한 아이템을 던전에서 찾아오기
- **전리품 판매 수입 → 루이제를 위한 구매**: 예) "이 목걸이 루이제에게 줄까?"
- **납치 이벤트 방지 조건**: 특정 보스를 쓰러뜨려야 루이제의 안전이 보장됨

### 4.2 전투 시스템 구현 (기술 리스크)

**문제:** 1인칭 자동 전투 + EH 상태이상/장비는 텍스트 어드벤처보다 무겁다.

**해결 방안:**
- workspace에 있는 `dungeon_adventure_system.md`와 `module_elderitch_horror_rpg.md`를 기반으로 스코프를 확정
- Phase 단위로 구현: 기본 자동 전투 → 장비 시스템 → 상태이상 → EH 다이스 풀
- PlaceScene과의 통합 설계: 던전도 "장소"로 취급 (`place_id: "dungeon_01"`)

### 4.3 게임 타이틀 (마케팅 리스크)

**문제:** "GuildMaster"는 현재 "모험가 연애 어드벤처" 컨셉과 불일치.

**해결 방안:** 타이틀 변경 검토. 후보:
- "Adventure Town" (이미 title_screen에 표시됨)
- "Owl's Nest" (세계관 중심)
- "Hotel Morgana" (이전 컨셉의 유산, 루이제의 여관 중심)
- "Princess Blitz: Side Story" (PPB 직접 연결)
