# 던전 탐험 & 이벤트 판정 시스템

> PbtA(Powered by the Apocalypse) 2d6 기반 '부분 성공' 텍스트 탐험

---

## 1. 개요

던전 탐험은 **텍스트 어드벤처 북 스타일**로 진행됩니다. 턴제 전투나 스탯 뻥튀기 없이, 매 이벤트마다 발생하는 상황을 **선택지 + 주사위 판정**으로 돌파합니다.

핵심 원칙:
- **플레이어는 영웅이 아님**: 괴물을 써는 히어로가 아니라, 도구와 기지로 척박한 환경을 간신히 살아남는 **생존형 모험가**
- **숫자 은닉**: 명시적인 성공 확률(%)을 보여주지 않음. 스탯과 아이템에 따라 **상황 묘사 텍스트**가 달라지며, 플레이어가 직접 유추
- **전투 최소화**: 전투는 "구색만 맞추는 수준". 생존 판단과 관계에 집중

---

## 2. PbtA 2d6 판정 시스템

던전월드 등 PbtA TRPG에서 사용하는 판정을 채택합니다.

### 2.1 판정 공식

```
[주사위 2개의 합(2d6)] + [스탯/아이템 보정치]
```

| 결과 | 구간 | 의미 |
|---|---|---|
| 🟢 **완전 성공** | 10 이상 | 아무 피해 없이 완벽하게 상황을 돌파 |
| 🟡 **부분 성공** | 7 ~ 9 | 돌파에는 성공하지만 **대가(Consequence)**를 치름 |
| 🔴 **실패** | 6 이하 | 돌파에 실패하고 큰 피해를 입거나 상황이 악화 |

### 2.2 부분 성공이 핵심인 이유

주사위 2개의 합은 **정규 분포(종모양)**를 그립니다. 7이 가장 높은 확률로 나옵니다.

- 플레이어는 **자원이 깎이며 간신히 살아남는** 경험이 가장 빈번함
- 이는 "평범하고 나약한 모험가의 척박한 현실"을 **확률적으로도, 서사적으로도** 가장 잘 대변함

## 3. 던전 이벤트 구조

### 3.1 이벤트 흐름

```
[상황 묘사 텍스트] → [선택지] → [2d6 판정] → [3중 분기 결과]
```

### 3.2 3중 분기 예시 (독가스실)

**🟢 10+ (완전 성공)**
> "숨을 꾹 참고 무사히 통과했다."

**🟡 7~9 (부분 성공)**
> "가스실을 통과하는 데는 성공했지만, 짙은 가스를 들이마셔 기침이 멈추지 않는다."
> → HP -5 / 정신력 -2

**🔴 6- (실패)**
> "지독한 독가스에 정신을 잃을 뻔했다. 황급히 뒤로 물러날 수밖에 없었다."
> → HP -10 / 탐색 진행도 하락

### 3.3 결과 다양성

단순한 HP 소모를 넘어, 다음과 같은 **대가**를 설계합니다:

| 대가 유형 | 예시 |
|---|---|
| 자원 소모 | HP, 정신력, 소모품(횃불, 회복약) |
| 상태 이상 | `add_condition("poisoned", 3)` → 마을에서 루이제가 걱정 |
| 미래 변형 | "발목을 삐끗했다 → 다음 이벤트는 강제로 '좁은 통로'" |
| 강제 귀환 | 체력 0이 아니어도 "너무 무서워서 도망친다" |

## 4. 성장 곡선: 레벨업 없는 진보

던전 탐험의 성장은 **'지식(경험)'과 '도구(장비)'의 누적**입니다.

```
[처음] 맨몸으로 독가스실에 진입 → 6- 실패 → 간신히 귀환
   ↓
[마을] 방독면 구입 (아이템)
   ↓
[재도전] 독가스실 이벤트 텍스트가 "방독면 덕에..."로 변경 → 7~9 또는 10+ 달성
```

- `has_item` 조건으로 이벤트 분기
- 보유 아이템에 따라 **이벤트 묘사 텍스트 자체가 달라짐**

## 5. Ink 구현 방법

### 5.1 외부 함수

Ink 스크립트에서 `~ roll_2d6(bonus)`를 호출합니다. Godot의 `InkActionBridge`에 바인딩되어 있습니다.

```ink
EXTERNAL roll_2d6(bonus)
EXTERNAL change_metric(key, amount)
EXTERNAL advance_time(units)
```

### 5.2 분기 예시

```ink
=== start ===
독가스가 퍼진 하수도 구간이다. 앞으로 나아갈까?

~ temp result = roll_2d6(0)

{ result >= 10:
    숨을 꾹 참고 무사히 통과했다.
    ~ advance_time(1)
    -> next_event
- result >= 7:
    가스실을 통과했지만 기침이 멈추지 않는다.
    ~ change_metric("player.hp", -5)
    ~ change_metric("player.sanity", -2)
    -> next_event
- else:
    지독한 독가스에 정신을 잃을 뻔했다.
    ~ change_metric("player.hp", -10)
    -> retreat
}
```

### 5.3 아이템 보정 예시

```ink
VAR has_gas_mask = false

=== start ===
{ has_gas_mask:
    방독면 덕에 독가스를 거뜬히 통과했다.
    ~ advance_time(1)
    -> next_event
- else:
    ~ temp result = roll_2d6(0)
    { result >= 10: ... }
}
```

## 6. 폴더 구조

```
data/dungeon_events/
├── sewer_gas.ink          # 하수도 독가스실
├── collapsing_trap.ink    # 붕괴하는 함정
├── monster_encounter.ink  # 몬스터 조우
└── *.ink.json             # 컴파일된 출력물
```

컴파일 명령:
```bash
tools/inklecate/inklecate.exe -o data/dungeon_events/sewer_gas.ink.json data/dungeon_events/sewer_gas.ink
```

## 7. 템플릿화 (Phase 2)

1인 개발에서 3중 분기는 **3배의 대사량**을 의미합니다. 다음 단계에서 `data/dungeon_templates/` JSON으로 카테고리별 공통 문구를 템플릿화합니다.

| 카테고리 | 템플릿 예시 |
|---|---|
| `gas_chamber` | "{place}에서 독가스를..." → 장소명만 치환 |
| `trap_floor` | "{place}의 바닥이 무너지기 시작한다..." |
| `monster_ambush` | "{creature}의 습격을 받았다..." |
