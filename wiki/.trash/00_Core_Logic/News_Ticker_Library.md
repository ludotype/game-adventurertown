# 뉴스 티커 라이브러리 (News Ticker Library)

## 1. 개요

뉴스 티커는 게임 내 이벤트를 **자극적인 언론 보도 스타일**로 변환하여 플레이어에게 전달하는 시스템입니다. 시스템 로그를 기반으로 동적으로 생성되며, 상황별 템플릿을 사용합니다.

---

## 2. 템플릿 구조

### 2.1. 기본 템플릿 포맷
```
[시간] [장소]에서 [주체]가 [행동]! [결과/여파] [추가 코멘트]
```

### 2.2. 감정/극성 태그
| 태그 | 사용 상황 | 예시 |
|------|----------|------|
| `[긴급]` | 사망/배신 등 중대 이벤트 | `[긴급] 모험가 사망... 길드 책임론 대두` |
| `[단독]` | 희귀/특별 이벤트 | `[단독] 전설의 보물 발견?` |
| `[속보]` | 전투 승리/패배 | `[속보] 던전 정복 성공!` |
| `[충격]` | 관계 파탄/스캔들 | `[충격] 길드원 간 불륜 의혹` |
| `[흥미]` | 일상적 재미 요소 | `[흥미] 주점에서의 난투극` |

---

## 3. 상황별 템플릿 모음

### 3.1. 사망 관련 (Death)

#### 전사/타격
```yaml
templates:
  - "[{actor}] 님이 [{dungeon}]에서 [{killer}]에게 최후를 맞이하셨습니다... 길드에 애도가 이어집니다."
  - "[{actor}], 영원한 안식으로. [{dungeon}]의 안개가 한 명의 모험가를 삼켰습니다."
  - "[충격] [{actor}] 사망! 동료 [{witness}]의 미흡한 대응이 원인?"
  - "[긴급] [{actor}] 님, 최후의 순간 [{last_words}]... 유언은 사실일까?"
  - "[{actor}], [{dungeon}]의 전설이 되다. 유품은 [{heir}]에게 상속."
```

#### 배신에 의한 사망
```yaml
templates:
  - "[충격] [{actor}]가 [{betrayer}]의 배신으로 사망... 길드 내 trust 크랙?"
  - "[{dungeon}]의 어둠 속, [{betrayer}]가 [{actor}]를 뒤에서 찔렀다는 제보!"
  - "[{actor}]의 죽음 뒤에 숨겨진 [{betrayer}]의 음모... 동기는 보물?"
```

---

### 3.2. 배신/도주 (Betrayal)

#### 전리품 독차지
```yaml
templates:
  - "[충격] [{betrayer}]가 파티를 버리고 [{loot}]를 훔쳐 달아났습니다!"
  - "[{dungeon}]에서 [{betrayer}]의 배신... 동료들만 눈물 흘리네."
  - "[{betrayer}], [{value}]골드의 보물을 들고 안개 속으로. 추적 가능할까?"
  - "[단독] [{betrayer}]의 배신은 계획된 것? 길드 정보 유출 의혹!"
```

#### 전투 중 도주
```yaml
templates:
  - "[{coward}]가 위기 상황에서 동료를 버리고 도주... 명예는 어디에?"
  - "[{dungeon}] 전투 중 [{coward}]의 단독 탈출, 남은 파티원 생사불명"
  - "[{victim}]을 구하지 않고 떠난 [{coward}], 길드 규정 위반 조사 예정"
```

---

### 3.3. 사랑/관계 (Love & Relationship)

#### 새로운 연애 시작
```yaml
templates:
  - "[속보] [{actor_a}]와 [{actor_b}], 던전에서 피어난 사랑? 목격담 속출!"
  - "[{actor_a}]가 [{actor_b}]에게 [{gift}] 선물... 주점에서의 데이트 목격!"
  - "[{actor_a}]♥[{actor_b}], 모험가 커플 탄생? 길드 내 환호!"
  - "[{actor_a}]와 [{actor_b}]의 밀회, [{location}]에서 포착!"
```

#### 삼각관계/질투
```yaml
templates:
  - "[충격] [{actor_a}]와 [{actor_b}] 사이 [{actor_c}]가 끼어든 삼각관계!"
  - "[{actor_c}]의 질투 폭발... [{actor_a}]와 [{actor_b}]의 사이를 의심?"
  - "주점에서 [{actor_a}]와 [{actor_b}]가 [{actor_c}] 앞에서 다툼! 원인은?"
```

#### 이별/관계 파탄
```yaml
templates:
  - "[{actor_a}]와 [{actor_b}], 드디어 결별? [{actor_a}]의 눈물 목격!"
  - "[{actor_b}]가 [{actor_a}]를 냉정하게 차버림... 이유는 성격 차이?"
  - "[{actor_a}]와 [{actor_b}]의 관계, 던전 스트레스가 원인?"
```

---

### 3.4. 전투/던전 (Combat & Dungeon)

#### 승리
```yaml
templates:
  - "[속보] [{party_leader}] 파티, [{dungeon}] 정복 성공! 획득 전리품 [{loot_value}]골드!"
  - "[{boss_name}] 격파! [{actor}]의 결정적 한 방이 전설이 되다."
  - "[{dungeon}] 최초 정복? [{party}]의 쾌거, 길드 역사에 기록될 만한!"
  - "[{actor_a}]와 [{actor_b}]의 콤보, [{boss_name}]를 무너뜨리다!"
```

#### 패배/위기
```yaml
templates:
  - "[긴급] [{party}] 파티, [{dungeon}]에서 대피... [{casualties}]명 부상"
  - "[{actor}]의 과오로 파티 전멸 위기? [{witness}]의 폭로!"
  - "[{dungeon}]의 [{monster}]에게 [{party}] 패퇴... 다음은?"
  - "[흥미] [{actor}]의 무모한 돌진, 파티를 위기에 빠뜨려"
```

---

### 3.5. 사회적 이벤트 (Social)

#### 주점/난투
```yaml
templates:
  - "[흥미] 어젯밤 [{location}]에서 [{actor_a}]와 [{actor_b}] 난투극!"
  - "[{actor_a}]와 [{actor_b}]의 주점 대결, 원인은 [{reason}]?"
  - "[{actor_a}], 술김에 [{actor_b}]에게 [{action}]... 사과 예정?"
  - "[{location}] 파손! [{actor_a}]와 [{actor_b}]의 난투로 손해배상 청구"
```

#### 상담/치료
```yaml
templates:
  - "[{actor}]가 심리 상담실 방문... 던전 트라우마?"
  - "[{actor}]와 [{actor_b}]의 관계 상담, [{counselor}]가 중재"
  - "[{actor}], 치료 중 [{healer}]와 교감... 새로운 인연 시작?"
```

---

### 3.6. 길드 운영 (Guild Management)

#### 고용/방출
```yaml
templates:
  - "[속보] 길드에 새로운 모험가 [{actor}] 합류! 특기는 [{specialty}]"
  - "[{actor}] 길드 방출... 이유는 [{reason}]?"
  - "[{actor}]가 길드를 자진 탈퇴, [{new_guild}]로 이적?"
  - "[단독] [{actor}]의 방출 뒤에 숨겨진 길드 내부 갈등"
```

#### 전리품 거래
```yaml
templates:
  - "[{actor}]가 [{item}]을 [{price}]골드에 판매... 시세는 적정?"
  - "[{rare_item}] 등장! [{actor}]의 행운, 길드 매출 UP"
  - "[단독] [{actor}]가 획득한 [{item}], 사실 저주받은 물건?"
```

## 4. 템플릿 변수 정의

| 변수명 | 설명 | 예시 값 |
|--------|------|---------|
| `[{actor}]`, `[{actor_a/b/c}]` | 캐릭터 이름 | "크로우", "엘라" |
| `[{dungeon}]` | 던전 이름 | "안개의 폐광", "잊힌 신전" |
| `[{location}]` | 장소 | "달빛 주점", "길드 상담실" |
| `[{killer}]` | 처치자 | "고블린 왕", "독 안개" |
| `[{loot}]`, `[{item}]` | 아이템 | "드래곤검", "1000골드" |
| `[{loot_value}]`, `[{price}]` | 가치 | "5000", "10000" |
| `[{value}]` | 금액 | "50000" |
| `[{party}]`, `[{party_leader}]` | 파티/리더 | "레오 파티" |
| `[{boss_name}]` | 보스명 | "타락한 기사" |
| `[{monster}]` | 몬스터 | "오크 무리" |
| `[{reason}]` | 사유 | "전리품 분배", "말다툼" |
| `[{witness}]` | 목격자 | "마을 사람" |
| `[{heir}]` | 상속인 | "동료 B" |
| `[{gift}]` | 선물 | "크리스탈 목걸이" |
| `[{counselor}]` | 상담사 | "안나" |
| `[{healer}]` | 힐러 | "미아" |
| `[{last_words}]` | 유언 | "동료들을 부탁해..." |
| `[{casualties}]` | 사상자 수 | "2" |
| `[{specialty}]` | 특기 | "화염 마법" |
| `[{new_guild}]` | 이적 길드 | "Golden Lion" |
| `[{action}]` | 행동 | "모욕" |
| `[{betrayer}]` | 배신자 | "잭" |
| `[{coward}]` | 겁쟁이 | "토드" |
| `[{victim}]` | 피해자 | "사라" |

## 5. 동적 변형 규칙

### 5.1. 관계도 반영
```yaml
# 호감도에 따른 표현 변화
relationship_modifier:
  high_affection:  # 호감도 > 50
    actor_a: "애인 [{actor_a}]"
    actor_b: "연인 [{actor_b}]"
  
  low_affection:  # 호감도 < -30
    actor_a: "숙적 [{actor_a}]"
    actor_b: "라이벌 [{actor_b}]"
  
  betrayal_history:
    prefix: "전 배신자"
    suffix: "(신뢰 복구 중)"
```

### 5.2. 성격 태그 반영
```yaml
# 성격에 따른 수식어
trait_modifier:
  무모한: "무모한 [{actor}]"
  탐욕스러운: "욕심 많은 [{actor}]"
  의리파: "의리파 [{actor}]"
  냉혈한: "차가운 [{actor}]"
  허세부리는: "허세쟁이 [{actor}]"
```

### 5.3. 빈도 제한
```yaml
spam_protection:
  same_actor_cooldown: 3  # 동일 인물 3턴 연속 등장 방지
  same_event_type_cooldown: 2  # 동일 유형 2턴 제한
  max_daily_ticker: 20  # 하루 최대 20개
```

## 6. 뉴스 티커 생성 파이프라인

```
[시스템 이벤트 발생]
    ↓
[이벤트 분류: 사망/배신/사랑/전투/사회/운영]
    ↓
[템플릿 후보군 선정 (3~5개)]
    ↓
[변수 매핑: actor, location, value 등 추출]
    ↓
[관계도/성격 태그 기반 수식어 적용]
    ↓
[중복/스팸 필터링]
    ↓
[최종 템플릿 선택 및 문자열 생성]
    ↓
[길드 마스터에게 푸시!]
```

## 7. 예시 출력

### 시나리오: 복합 이벤트
```
이벤트 체인:
1. [전투] 레오(무모한)가 돌격 → 미아(신중한)가 구출 → 호감도 +25
2. [발견] 레전드 검 발견
3. [분쟁] 잭(탐욕스러운)이 독차지 시도
4. [배신] 잭이 도주
5. [사망] 레오가 추격 중 사망
6. [연애] 미아가 레오의 유품을 지킴 → 새로운 관계 맺음
```

### 생성된 뉴스 티커 시퀀스
```
[1] [속보] 레오 파티, 안개의 폐광에서 레전드 검 발견! 추정 가치 100,000골드!

[2] [충격] 잭이 레전드 검을 들고 파티를 버림! 무모한 레오 추격 중 사망...

[3] [속보] 의리파 미아, 레오의 유품을 지키겠다 선언. 두 사람의 특별한 인연?

[4] [단독] 잭의 배신 뒤에 숨겨진 욕심의 전과? 길드 정보 유출 가능성

[5] [충격] 레오, 안개의 폐광의 전설이 되다. "동료들을 부탁해..." 마지막 유언
```
