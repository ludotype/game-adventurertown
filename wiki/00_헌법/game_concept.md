# Guild Master - 게임 컨셉트 문서

> **타겟 플랫폼**: PC (Steam) / Mobile (iOS, Android)
> **개발 인원**: 1인 (아티스트/개발자)
> **장르**: 1인칭 판타지 연애 어드벤처 + 텍스트 어드벤처 북
> **IP**: 프린세스 블리츠(PPB) 스핀오프

---

## 1. 비전 (Vision)

**"위험지대의 마을에서 모험가로 살아가며, 한 사람의 마음을 얻어라"**

플레이어는 아울네스트라는 위험지대 마을의 모험가가 됩니다. 낮에는 텍스트 어드벤처 스타일의 던전을 탐험하고, 저녁에는 주점에서 사람들과 어울리며, 반복되는 일상 속에서 루이제의 마음을 얻어가는 것이 이 게임의 핵심입니다.

### 핵심 경험
- **연애 이벤트 중심의 스토리**: 모든 게임플레이가 관계 진전으로 연결
- **단계적 호감 상승의 몰입**: 무관심 → 관심 → 친밀의 자연스러운 변화
- **텍스트 어드벤처의 긴장감**: 던전에서의 선택이 생존과 직결
- **아티스틱 몰입**: 고품질 캐릭터 CG로 전달되는 감정적 순간들

---

## 2. 핵심 컨셉 (Core Concepts)

### 2.1 게임 플레이 핵심 루프

```
[마을 일상 (대화/행동)] → [던전 탐험 (텍스트 어드벤처)] → [귀환/결산] → [주점에서의 시간] → [반복]
```

플레이어는 모험가입니다. 루이제는 여관 주인입니다. 직접 싸우는 영웅이 아니므로, 전투보다는 관계에 집중합니다.

### 2.2 차별화 포인트

| 기존 연애 게임 | Guild Master의 차별화 |
|---------------|----------------------|
| 선택지로 호감도 상승 | 행동 전체(시간 배분, 대화, 탐험 귀환 시각)가 관계에 영향 |
| 히로인 루트 분기 | 제한된 엔딩 대신, 호감 단계별 이벤트의 지속적 축적 |
| 전투 시스템 분리 | 텍스트 어드벤처 북 스타일로 VN 인터페이스와 자연 통합 |
| 루트 진입 후 타 캐릭터 배제 | 관계 상태가 지속되며, 행동이 누적 효과를 가짐 |
| 단일 히로인 고정 | 루이제(MVP)를 시작으로 단계적 캐릭터 확장 |

### 2.3 3가지 기둥 (The Three Pillars)

1. **관계 (Relationship)**
   - 루이제(첫 MVP 히로인)와의 단계적 호감 상승 (무관심 → 관심 → 친밀)
   - 호감도에 따른 대사/이벤트 분기
   - 마을 NPC들과의 교류가 루이제와의 관계에 간접 영향
   - **확장**: 동일한 시스템으로 추가 히로인/캐릭터를 단계적으로 도입 예정

2. **탐험 (Adventure)**
   - 텍스트 어드벤처 북 스타일의 던전 탐험
   - VN 엔진의 메시지 창 + 선택지로 자연스럽게 구현
   - 장비/상태 확인, 귀환 시각 판단이 핵심 전략

3. **일상 (Daily Life)**
   - 시간대별 NPC 위치 변화
   - 장소 이동, 행동 선택으로 시간 소비
   - 매일의 작은 선택이 누적되어 관계 변화

## 3. 게임 플레이 상세 (Gameplay Details)

### 3.1 루이제와의 연애 시스템 (MVP 히로인)

루이제는 첫 번째로 구현되는 MVP 히로인입니다. 동일한 affection 시스템과 Ink 대화 구조로 추가 캐릭터를 단계적으로 확장할 예정입니다.

**초기 단계 (무관심)**
- 루이제는 바쁘고 주인공에게 큰 신경을 쓰지 않음
- 자신에 대한 얘기도 별로 하지 않음
- 예: *"...오셨어요? 음료는 저기 메뉴판 보고 주문하세요."* (시선도 마주치지 않음)

**중간 단계 (관심)**
- 쉬는 시간에 펍에 들러 말을 걸거나 일을 돕다 보면 신경을 더 씀
- 처음에는 큰 인상을 남기지 않음
- 예: *"응? 도와주겠다고? 그럼 여기 이 냄비가 타지 않게 좀 젓고 있어 줄래? 아, 바쁘다 바칠!!"* (휙 가 버림)

**후반 단계 (친밀)**
- 그녀의 의뢰를 최우선으로 처리하며 도움이 되면 변화가 생김
- 예: *"응, 주인공? 지금 나가는 거야? 조심해~"* 라며 지나갈 때 웃으며 손을 흔들어 줌

### 3.2 던전 탐험 시스템

**텍스트 어드벤처 북 스타일**
```
당신의 앞에는 어두운 하수도가 펼쳐져 있다...
HP: 90/100 | 정신: 20/20 | 탐색도: 2/10
선택지: [나아간다] [돌아간다]

[나아간다]
한 무리의 래틀링들이 보인다. 어떻게 할까?
HP: 90/100 | 정신: 20/20 | 탐색도: 3/10
선택지: [돌진한다] [숨어서 살핀다] [후퇴한다]
```

- VN 엔진의 메시지 창과 선택지로 자연스럽게 처리
- 각 이벤트 모듈 = 텍스트 + 선택지 + 간단한 수치 변화(HP/정신/탐색도)
- 장비와 상태를 확인하며 "언제 돌아갈지"를 판단하는 것이 핵심 전략
- 구색을 맞추는 정도의 최소화된 전투. 플레이어는 영웅이 아님

### 3.3 시간/장소 시스템

- 24시간제 시간 진행. 행동마다 시간 소비
- 시간대별 NPC 위치 변화 (스케줄 시스템)
- 장소 간 이동 + 장소별 행동 선택지
- **매 턴(장소 진입/시간 경과)마다 화면 좌측에 정경 텍스트 갱신**
  - 장소 묘사: `description` 또는 `descriptions[time_of_day]`
  - 플레이어 상태 묘사: HP, 정신, 배고픔 등 MetricStore 기반
  - NPC 묘사: 메인 NPC(상호작용 가능) + 서브 NPC(배경 인물)
- 마을 지도: 여관 → 복도 → 로비 → 거리 → 광장 → 주점/상점

### 3.4 호감도 시스템

- MetricStore 기반으로 `npc.luise.affection` 등 관리
- 호감도에 따른 interaction 분기 (ConditionEvaluator)
- 대화/행동으로 호감도 변화, 조건부 이벤트 발생

## 4. 아트 방향성 (Art Direction)

### 4.1 비주얼 스타일

**배경**
- 장소별 배경 일러스트 (여관, 주점, 거리, 광장, 상점 등)
- 던전: 텍스트 묘사 + 분위기 이미지 (최소화)

**캐릭터**
- **메인 NPC CG**: 고품질 캐릭터 스탠딩 (표정 3~5종)
  - 루이제 (여관 주인, 첫 MVP 히로인)
  - 주요 상인/퀘스트 제공자 2-3명
  - 추가 히로인 (단계적 확장)
- **모험가 NPC**: 심플 스프라이트
- **SCG 시스템**: 대화 중 캐릭터 초상화 교체 `# scgc=animation_character` 태그 (Ink)

**UI**
- **테마**: 네오-클래식 판타지
- VN 인터페이스 기반 (대화 창 + 선택지)
- 장소 화면: 배경 + NPC 오버레이 + 행동 패널 + 이동 바 + **정경 텍스트 패널(좌측 상단)**
- **정경 텍스트 패널**: 배경 투명도 10%, 폰트 22px + 2px 검은 아웃라인, 장소/시간/플레이어 상태/NPC 묘사를 매 턴 갱신

### 4.2 스크린샷 컨셉


## 5. 기술 스택 (Technical Stack)

### 5.1 엔진 선택: Godot 4 + Ink

**선택 이유**
1. **1인 개발 최적화**: GDScript 학습 곡선 완만, 빠른 프로토타이핑
2. **Ink 내러티브 엔진**: 검증된 컴파일러, 변수/조건, Inky 에디터 지원
3. **데이터 드리븐 아키텍처**: JSON 폴더 스캔으로 콘텐츠 추가
4. **멀티플랫폼**: PC + Mobile 동시 빌드 가능

### 5.2 아키텍처 개요

- **Auto-Discovery**: JSON을 폴더에 추가하면 자동 등록
- **ID-First Referencing**: 모든 엔티티는 고유 ID로 참조
- **Definition ≠ State**: JSON(정의)은 읽기 전용, 런타임 상태는 별도
- **Vocabulary, not Table**: 행동/조건을 작은 어휘로 조합
- 자세한 내용은 `architecture.md` 참조

## 6. 엔딩 시스템

**제대로 된 엔딩은 필요 없음.**
- 단순히 루이제와 친해져서 그 상태를 유지하든가, 게임을 새로 시작하든가. 플레이어 마음대로.
- 제작자가 원하면 각 호감 단계에서의 이벤트를 언제든 추가해서 내용을 늘릴 수 있음.
- **연애 시스템 템플릿화**: 이 시스템은 후속작이나 다른 게임의 연애 시스템을 제작할 때 좋은 템플릿이 됨.

## 7. 스코프 관리 원칙 (Scope Management)

### 7.1 하지 않을 것 (Won't Do)

- ❌ **실시간/턴제 RPG 전투**: 텍스트 어드벤처 북으로 대체
- ❌ **멀티플레이**: 1인 싱글 플레이어
- ❌ **3D 그래픽**: 2D + VN 인터페이스
- ❌ **풀 보이스**: 텍스트만, 주요 이벤트만 CG
- ❌ **길드 관리/경영 시뮬레이션**: 관리자가 아닌 모험가 1인
- ❌ **자동 전투/관전**: 플레이어가 직접 선택지로 진행
- ❌ **건설/시설 업그레이드**: 마을 건설 없음
- ❌ **평판 시스템**: 길드 평판 없음

### 7.2 단순화 원칙

| 대상 | 복잡한 버전 | 단순화 버전 |
|------|------------|------------|
| 전투 | 턴제 RPG | 텍스트 선택지 + 수치 변화 |
| 이동 | 월드 맵 탐험 | 장소 간 이동 (노드 기반) |
| 관계 | 다수 히로인 루트 | 루이제 기반 MVP 히로인 + 단계별 캐릭터 확장 |
| 스토리 | 엔딩 분기 | 엔딩 없음, 상태 유지 |

### 7.3 콘텐츠 파이프라인

**데이터 드리븐**: 새로운 장소, NPC, 대화, 이벤트는 JSON 추가만으로 생성

## 8. IP 전략

이 게임은 '프린세스 블리츠'의 스핀오프. 루이제는 원래 게임에서 등장하는 주요 NPC. 기존 게임 프로젝트가 사보타주를 당한 지금, 이런 방식으로 세계관과 루이제라는 캐릭터를 사람들에게 익숙하게 만드는 것이 마케팅/생존 전략.
