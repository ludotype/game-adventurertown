# 코딩 및 작업 규칙

> **본 문서는 프로그래머(Claude 포함)가 반드시 준수해야 하는 규칙입니다.**
>
> Claude는 작업 시작 시 이 문서를 확인하고, 위반 시 즉시 수정합니다.

---

## 1. 파일 수정 규칙

### 1.1 Read 먼저
- 기존 파일을 **반드시 읽고 전체 내용을 확인**한 후 수정합니다.

### 1.2 Edit 우선
- `Write` 대신 `Edit`를 사용합니다. 기존 내용 보존이 최우선입니다.

### 1.3 전체 교체 금지
- `Write`로 통째로 덮어쓰지 않습니다.
- 예외: 완전한 재작성이 필요한 경우 사용자 확인이 필수입니다.

### 1.4 끝까지 읽기
- 파일 끝부분(중요 사항, 공통 코드 등)까지 확인 후 수정합니다.

---

## 2. 문서 작성 규칙

### 2.1 언어
- **영어 알파벳(A-Z, a-z) 또는 한글만** 사용합니다.
- 중국어(汉字), 일본어(ひらがな/カタカナ/漢字), 기타 문자 사용 금지.

### 2.2 경로
- 사용자 요청 메모리는 **반드시 `./Agent/memory/`**에 저장합니다.
- 전역 경로(`~/.claude/...`, `C:\Users\...\.claude\...`) 사용 금지.

### 2.3 프로젝트 문서 동기화
- 새 스크립트/메카닉 변경 시 `wiki/planner-guide.md` 관련 섹션도 자동 갱신합니다.
- 기획자(비프로그래머)가 이해할 수 있는 언어로 작성합니다.

---

## 3. 메모리 시스템 규칙

### 3.1 자동 메모리 (Claude Automatic)
- `~/.claude/projects/<project>/memory/`에 저장
- Claude가 자동으로 관리

### 3.2 사용자 요청 메모리 (Explicit Memory)
- **반드시 `./Agent/memory/`** 사용
- 파일명: `[type]_[topic].md`
- 타입: `user_`, `feedback_`, `project_`, `reference_`
- 인덱스: `./Agent/memory/MEMORY.md` 반드시 유지

### 3.3 저장 절차 (사용자가 "기억해"라고 했을 때)
1. MEMORY.md 읽기 → 기존 파일 확인
2. 중복/충돌 체크
3. 사용자 선택 (갱신/병합/추가/무시)
4. 저장 후 MEMORY.md 인덱스 업데이트

---

## 4. Git 커밋 규칙

- **새 커밋 생성** (amend 금지, 단 사용자 요청 시 예외)
- Claude Code + Happy Co-Authored-By 크레딧 포함
- `.env`, credential 파일 등 민감 파일 커밋 금지

---

## 5. 안전 규칙

- 파괴적 명령(`rm -rf`, `git reset --hard` 등)은 사용자 확인 후 실행
- 공유 시스템(PR, push 등)은 사전 확인
- 외부 서비스 업로드는 사전 승인 필요
