선생, 기획자에게 그대로 전달해도 되게 정리해 둘게.

---

## 1. 무엇이 자동으로 기록되는지

플레이어가 스토리에서 **메인 NPC 캐릭터 CG**가 화면에 뜰 때마다, 게임 시간(`total_minutes`와 같은 축)이 아래 변수에 **덮어쓰기**로 저장돼.

| NPC | 저장되는 전역 변수 키 |
|-----|----------------------|
| 세하 | `scg.last_seen.seha` |
| 노루 | `scg.last_seen.noru` |
| 태민 | `scg.last_seen.taemin` |
| 준호 | `scg.last_seen.junho` |
| 유진 | `scg.last_seen.yujin` |

**조건:** 스크립트가 `character seha.…`처럼 **맨 앞 토큰이 위 다섯 이름**일 때만 갱신돼. (다른 캐릭터 CG는 이 변수들을 건드리지 않음.)

---

## 2. 트리거 JSON에서 “SCG 본 뒤 X분 지난 뒤에만” 쓰는 법

`event_triggers.json`, `incomingcall_triggers.json`, `outgoingcall_triggers.json`의 **이벤트 하나의 `conditions` 배열**에, 아래처럼 한 줄을 추가하면 돼.

**예: 세하 CG를 본 지 30분이 지난 뒤에만 이 이벤트가 될 수 있게**

```json
{ "key": "scg.last_seen.seha", "op": "elapsed>=", "value": "30" }
```

의미를 풀면:

- `key`: “마지막으로 세하 CG가 뜬 시각”이 들어 있는 변수  
- `op`: “지금 시각 − 그 시각”이 몇 분인지 계산해서 비교  
- `value`: “그래도 되려면 최소 몇 분은 지나야 하는지” (숫자, 분 단위)

`value` 자리에 다른 전역 변수 이름을 쓰면, 그 변수 값으로 치환되는 방식은 **기존 트리거와 동일**해. (예: `"value": "total_minutes"`처럼 쓰던 것과 같은 규칙.)

---

## 3. 쓸 수 있는 연산자 목록

같은 `key`에 대해 아래 `op`를 쓸 수 있어.

| op | 의미 (대략) |
|----|-------------|
| `elapsed>=` | “SCG 본 뒤 **최소** N분 지남” → **N분 이후에만** 이벤트 가능할 때 |
| `elapsed>` | N분 **초과** |
| `elapsed<=` | “**N분 이내**에 CG를 봤을 때” 같은 제한 |
| `elapsed<` | N분 **미만** |
| `elapsed==` | 정확히 N분 차이 (실제로는 잘 안 쓰일 수 있음) |
| `elapsed!=` | N분이 아닐 때 |

가장 많이 쓸 건 **`elapsed>=`** (쿨다운·몰입용)와 **`elapsed<=`** (방금 대화한 직후만 쓰는 이벤트)야.

---

## 4. “아직 한 번도 CG 기록이 없을 때” 동작

해당 NPC에 대해 **`scg.last_seen.…`가 없거나 숫자가 아니면** “오래 전에 본 것과 같다”고 간주해.

- **`elapsed>=` / `elapsed>`** → **통과(true)**  
  → 예전 세이브나 신규에도, “30분 지난 뒤에만” 같은 조건이 **막히지 않게** 하려고 이렇게 잡혀 있어.  
- **`elapsed<=` / `elapsed<` / `elapsed==`** → **대부분 실패에 가깝게** 동작해.  
  → “방금 CG 봤을 때만” 같은 조건은, **한 번이라도 CG가 뜬 뒤**부터 의미가 생겨.

기획할 때는 **“막을 조건”은 `elapsed>=`**, **“직후만”은 `elapsed<=`** 조합이 직관적이야.

---

## 5. 스크립트(대사 분기 등)에서 “몇 분 전에 봤는지” 쓰는 법

스토리 쪽에는 **전역 변수 숫자**만 있으면 되고, 이미 있는 **`calculate`**로 빼면 돼.

예: “세하 CG 이후 경과 분”을 임시 변수에 넣기:

```text
calculate seha_scg_elapsed {global.total_minutes} - {global.scg.last_seen.seha}
```

- `scg.last_seen.seha`가 **아직 없으면** 기존 규칙상 `0`처럼 잡혀서, 계산 결과가 커질 수 있어.  
  → “정확히 몇 분 전”이 중요하면, 트리거용 `elapsed` 조건과 맞춰서 **먼저 CG가 한 번 뜨는 구간**을 두는 게 안전해.

## 6. 기획자에게 한 줄 요약

**“메인 다섯 NPC CG가 뜰 때마다 `scg.last_seen.{이름}`이 갱신되니, 이벤트 조건에 `elapsed>=` 같은 줄만 넣으면 ‘CG 본 지 N분 후에만’ 또는 ‘N분 안에만’을 JSON으로 직접 조절할 수 있다.”**

이 정도면 실무에서 바로 쓸 수 있을 거야, 선생.
