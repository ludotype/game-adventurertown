# Guild Master - 게임 컨셉트 문서

> **타겟 플랫폼**: PC (Steam) / Mobile (iOS, Android)  
> **개발 인원**: 1인 (아티스트/개발자)  
> **개발 기간**: 3-4개월 (MVP → 얼리 액세스)  
> **장르**: 관리형 전략 시뮬레이션 + 자동 전투 RPG

---

## 1. 비전 (Vision)

**"모험가들을 고용하고 키워서 마을을 발전시키는 길드 마스터가 되어라"**

플레이어는 마을의 길드 길드마스터가 되어 모험가들을 고용하고, 장비를 지원하며, 마을 시설을 업그레이드하는 관리자가 됩니다. 모험가들은 자율적으로 월드 맵을 탐험하고 몬스터를 처치하여 전리품을 가져오며, 플레이어는 이를 통해 길드를 성장시키고 주변 지역을 개척해 나갑니다.

### 핵심 경험
- **간접 컨트롤의 쾌감**: 직접 싸우지 않고 "육성"과 "관리"를 통해 결과를 만드는 전략적 재미
- **반복 플레이의 중독성**: 프로시저럴 생성으로 매번 새로운 맵과 모험가, 이벤트
- **성장의 가시성**: 마을이 판잣집에서 성벽 도시로 진화하는 과정
- **아티스틱 몰입**: 메인 NPC 캐릭터 CG를 통한 스토리텔링

## 2. 핵심 컨셉 (Core Concepts)

### 2.1 게임 플레이 핵심 루프

```
[길드 운영] → [모험가 고용/장비] → [자동 탐험] → [전투/수확] → [전리품 판매] → [시설 확장] → [반복]
```

플레이어는 관리자입니다. 모험가들은 자율적 에이전트입니다.

### 2.2 차별화 포인트

| 기존 게임 | Guild Master의 차별화 |
|-----------|----------------------|
| 루프 히어로 (Loop Hero) | 길드 운영 + 경제 시뮬레이션 추가, 단순 전투를 넘어선 관리 메타 |
| 아이들 히어로즈 (Idle Heroes) | 자동 전투는 유사하되, "길드"라는 거점 개념과 마을 건설 추가 |
| 다키스트 던전 | 간접 컨트롤은 유사하되, 더 밝은 톤과 간결한 시스템, 1인 개발 스코프 최적화 |

### 2.3 3가지 기둥 (The Three Pillars)

1. **탐험 (Exploration)**
   - 프로시저럴 생성 월드 맵
   - 안개(Shroud)가 가려진 미지의 영역 개척
   - 다양한 지형(숲, 동굴, 늪, 던전)과 환경 효과

2. **전투 (Combat)**
   - 턴제 자동 전투 (Turn-based Auto Battle)
   - 스피드 컨트롤 (1x → 8x 홀드 액셀러레이션)
   - 포지션/진영 시스템 (전열/후열)
   - 역할별 AI (탱커, 딜러, 힐러, 버퍼)

3. **경영 (Management)**
   - 모험가 고용/해고/육성
   - 장비 제작/구매/강화
   - 마을 시설 건설 (대장간, 주점, 병원, 마법사 탑 등)
   - 외부 세력과의 무역

## 3. 게임 플레이 상세 (Gameplay Details)

### 3.1 길드 운영 시스템

**모험가 시스템**
- **등급**: E → D → C → B → A → S (등급별 성장 한도 차이)
- **특성**: 각 모험가는 랜덤 특성 보유 ("야행성: 밤 전투시 공격력 +10%", "욕심쟁이: 전리품 20% 더 가져감")
- **스탯**: 힘(STR), 민첩(DEX), 지능(INT), 체(CON), 속도(SPD)
- **컨디션**: 피로도, 사기, 부상 상태 관리 필요

**고용/해고**
- 주점에서 랜덤 등장하는 모험가 스카우트
- 계약금 월급 개념 (시간 경과에 따른 유지비)
- S급 모험가는 특별 의뢰 완료 후 영입 가능

### 3.2 전투 시스템

**자동 전투 메커닉**
- **턴 순서**: SPD 스탯 기반 (높을수록 선공)
- **AI 우선순위**: 역할별로 다른 타겟팅/스킬 선택 로직
  - 전사: 전열 적, 낮은 HP 우선
  - 마법사: 후열 적, 높은 위협도 우선
  - 힐러: 아군 HP 50% 이하 시 힐 우선
- **진영(Formation)**: 4인 파티 (전열 2 / 후열 2)
  - 전열: 방어력 +20%, 받는 피해 -20%
  - 후열: 마법 공격력 +10%, 물리 공격력 -10%

**스피드 컨트롤**
- 홀드 버튼으로 속도 조절: 1x → 2x → 4x → 8x
- 고속 시 시각 효과 간략화 (LOD)
- 핵심 순간(치명타, 보스 킬) 자동 슬로우 모션

**전투 결과물**
- 경험치 (레벨업)
- 전리품 (장비, 재료, 희귀 아이템)
- 지도 정보 (미개척 지역 해금)

### 3.3 맵 탐험 시스템

**프로시저럴 생성**
- **월드 시드**: 매 게임마다 새로운 지형 생성 (퍼린 노이즈 기반)
- **POI (Point of Interest)**: 고정된 던전 입구, 마을, 특수 이벤트 지점
- **안개 제거**: 모험가가 이동하며 주변 타일 밝힘

**지형 타입**
| 지형 | 특징 | 전투 보정 |
|------|------|----------|
| **평원** | 이동 빠름, 적 보통 | 없음 |
| **숲** | 시야 제한, 은신 가능 | 궁수 공격력 +10% |
| **산** | 이동 느림, 높은 적 레벨 | 전사 방어력 +10% |
| **늪** | 이동 매우 느림, 독/질병 | 지속 피해 위험 |
| **던전** | 고정 맵, 보스 존재 | 특수 전리품 드랍 |

**AI 탐험 패턴**
- **탐험 모드**: 미밝힌 타일 우선 이동
- **사냥 모드**: 적 처치 우선
- **회수 모드**: 체력 30% 이하 시 귀환
- **의뢰 모드**: 특정 POI 향해 이동

### 3.4 경제/건설 시스템

**수익 구조**
- 모험가가 가져온 전리품 판매 (자동 또는 수동 선택)
- 길드 등급에 따른 정부 보조금
- 던전 발견 보너스

**지출 구조**
- 모험가 고용비/월급
- 시설 유지비
- 장비 구매/제작 비용
- 마을 방어 시설 (몬스터 웨이브 대비)

**건설 가능 시설**
| 시설 | 효과 | 최대 레벨 |
|------|------|----------|
| **주점** | 모험가 회복 속도 증가, 고용 가능 인원 증가 | 5 |
| **대장간** | 장비 제작, 강화, 수리 | 5 |
| **병원** | 부상 치료, 최대 체력 증가 버프 | 3 |
| **마법사 탑** | 마법 장비 인챈트, 스크롤 제작 | 3 |
| **무역소** | 외부 상인 접근, 희귀 아이템 구매 | 3 |
| **성벽** | 마을 방어력 (몬스터 습격 방어) | 3 |

## 4. 아트 방향성 (Art Direction)

### 4.1 비주얼 스타일

**월드/맵**
- **타일 기반 2D**: 32x32 픽셀 타일셋
- **미니멀 컬러 팔레트**: 기본 타일은 채도 낮은 색상, 중요 포인트는 채도 높게
- **안개 효과**: 미탐험 지역은 어둡게, 탐험한 곳은 밝게 (그라데이션)

**캐릭터**
- **메인 NPC**: 고품질 캐릭터 CG (사용자가 직접 그림)
  - 길드 마스터 (플레이어 캐릭터)
  - 주요 상인/퀘스트 제공자 3-4명
- **모험가**: 클래스별 심플 스프라이트 (12x24 픽셀)
  - 전사, 마법사, 궁수, 도적, 힐러 등 클래스 구분
- **적**: 종족별 스프라이트 (고블린, 오크, 스켈레톤, 늑대 등)

**UI**
- **테마**: 네오-클래식 판타지
- **레이아웃**: 인포그래픽 스타일 - 정보 밀도 높게, 한 화면에 모든 정보
- **애니메이션**: 최소화, 기능적 트랜지션만 사용 (0.2s 이내)

## 5. 기술 스택 (Technical Stack)

### 5.1 엔진 선택: Godot 4

**선택 이유**
1. **1인 개발 최적화**: GDScript 학습 곡선 완만, 빠른 프로토타이핑
2. **2D 타일 게임에 최적**: TileMap 시스템 우수
3. **멀티플랫폼**: PC + Mobile 동시 빌드 가능
4. **라이선스**: 무료, 로열티 없음
5. **용량**: 엔진 자체 경량, 모바일 설치 용량 작음

### 5.2 프로젝트 구조

```
res://
├── autoload/          # 싱글톤 (GameManager, SaveManager)
├── systems/           # 핵심 시스템
│   ├── combat/        # 전투 시스템
│   ├── economy/       # 경제/자원 관리
│   ├── exploration/   # 맵 생성, 탐험 AI
│   └── guild/         # 길드 관리
├── entities/          # 게임 오브젝트
│   ├── adventurers/   # 모험가 데이터/스프라이트
│   ├── enemies/       # 적 데이터
│   └── tiles/         # 맵 타일셋
├── ui/                # UI 씬/스크립트
├── scenes/            # 게임 씬
│   ├── main_game.tscn # 메인 루프
│   ├── combat.tscn    # 전투 씬
│   └── town.tscn      # 마을 관리 씬
└── resources/         # 데이터 테이블
    ├── items.json
    ├── skills.json
    └── events.json
```

### 5.3 데이터 관리

- **설정 데이터**: JSON/CSV (밸런스 조정용)
- **세이브 데이터**: Godot ConfigFile 또는 JSON
- **프로시저럴 생성**: 시드 기반, 재현 가능

## 6. 개발 로드맵 (Development Roadmap)

### Phase 1: 핵심 루프 (Week 1-2)
**목표**: 플레이 가능한 최소 단위 (MVP)

- [ ] 타일 기반 맵 이동 시스템
- [ ] 1개 클래스 모험가 (전사만)
- [ ] 간단한 자동 전투 (공격만)
- [ ] 길드 → 탐험 → 귀환 → 보상 수금 루프
- [ ] 기본 UI (HP바, 인벤토리)

**검증 기준**: "한 모험가가 길드를 나가서 적을 잡고 돈을 가져오는 것"이 5분 안에 가능

### Phase 2: 전투 완성 (Week 3-4)
- [ ] 4직업 클래스 (전사, 마법사, 궁수, 힐러)
- [ ] 턴제 전투 시스템 구현
- [ ] 스킬 시스템 (3개 스킬 per 클래스)
- [ ] 스피드 컨트롤 (1x/2x/4x)
- [ ] 포지션/진영 시스템
- [ ] 전투 이펙트 (데미지 라벨, 흔들림)

### Phase 3: 경제/건설 (Week 5-6)
- [ ] 장비 시스템 (제작, 강화)
- [ ] 마을 건물 4종 (주점, 대장간, 병원, 무역소)
- [ ] 자원 관리 (골드, 재료, 명성)
- [ ] 상점 UI
- [ ] 모험가 고용/해고

### Phase 4: 콘텐츠/반복성 (Week 7-8)
- [ ] 프로시저럴 맵 생성 (퍼린 노이즈)
- [ ] 랜덤 모험가 생성 (이름, 특성, 스탯 분포)
- [ ] 랜덤 이벤트 시스템 (12개 기본 이벤트)
- [ ] 지형 타입 4종 (평원, 숲, 산, 늪)
- [ ] 10종 적 몬스터

### Phase 5: 폴리시 (Week 9-10)
- [ ] 밸런싱 (경제 흐름, 난이도 커브)
- [ ] 모바일 최적화 (터치 UI, 성능)
- [ ] 사운드/BGM (8bit 스타일)
- [ ] 튜토리얼
- [ ] 버그 수정, 세이브/로드 완성

### Phase 6: 런칭 준비 (Week 11-12)
- [ ] 메인 NPC CG 삽입
- [ ] 스토리 이벤트 (간단한 시나리오)
- [ ] 업적 시스템
- [ ] 스토어 페이지 준비
- [ ] 얼리 액세스 출시

## 7. 스코프 관리 원칙 (Scope Management)

### 7.1 하지 않을 것 (Won't Do)

- ❌ **실시간 액션 전투**: 턴제 자동으로 한정
- ❌ **멀티플레이**: 1인 싱글 플레이어
- ❌ **3D 그래픽**: 2D 타일로 한정
- ❌ **풀 보이스**: 텍스트만, 주요 이벤트만 CG
- ❌ **오픈 월드**: 그리드 기반 타일 맵
- ❌ **플레이어 직접 전투**: 관리자 역할 유지

### 7.2 단순화 원칙

| 대상 | 복잡한 버전 | 단순화 버전 |
|------|------------|------------|
| 전투 | 실시간 스킬 사용 | 완전 자동, 스피드만 조절 |
| 이동 | 경로 설정 | AI가 알아서 탐험 |
| 건설 | 자유 배치 | 슬롯 기반 업그레이드 |
| 스토리 | 브랜칭 다이얼로그 | 이벤트 카드 형식 (선택지 2개) |
| 애니메이션 | 스프라이트 애니메이션 | 위치 이동 + 플래시 효과만 |

### 7.3 콘텐츠 파이프라인

**데이터 드리븐**: 새로운 적, 아이템, 이벤트는 JSON 추가만으로 생성

```json
{
  "id": "goblin_scout",
  "name": "고블린 정찰병",
  "hp": 25,
  "attack": 8,
  "drops": [" rusty_sword", "small_gold"],
  "ai_type": "aggressive"
}
```

## 8. 핵심 지표 (Success Metrics)

### 8.1 개발 지표
- **Day 1 Retention (테스트)**: 60%+
- **세션 길이**: 평균 15분
- **성장 사이클**: 30분마다 "업그레이드" 체감
