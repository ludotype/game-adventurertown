# 중요한 실패로부터의 교훈 자동 기록

## 규칙
작업 중 **중요한 실패**가 발생하고 그 원인/해결책을 파악했을 경우, **자동으로 메모리에 기록**한다.

## "중요한" 실패 기준
다음 조건 중 하나 이상에 해당하면 자동 기록 대상:

1. **도구 사용법 관련**: 특정 도구(Glob, Grep 등)의 예상치 못한 동작
2. **플랫폼 차이**: Windows/Linux/Mac 간 동작 차이
3. **반복 가능성**: 동일 실패가 재발할 가능성이 높음
4. **프로젝트 전반 영향**: 작업 방식 전체에 영향을 미치는 기본 설정

## 기록 내용
- 실패 증상 (무엇이 안 됐는가)
- 원인 분석 (왜 실패했는가)
- 해결책/올바른 방법
- 향후 적용 방식

## 저장 위치
- 타입: `feedback`
- 파일명: `feedback_[주제].md`
- 경로: `Agent/memory/`

## 예시 (이번 사례)
- **실패**: Glob 패턴 `"./Agent/memory/**/*.md"`가 파일을 찾지 못함
- **원인**: Windows에서 leading `./` 처리 불가
- **해결**: `"Agent/memory/**/*.md"`로 변경
- **기록**: `feedback_glob_patterns.md` 생성

**Why:** 사용자 제안 - 중요한 교훈은 능동적으로 기록하여 반복 실패 방지

**How to apply:** 실패 원인 파악 후 즉시 메모리 파일 생성 및 MEMORY.md 인덱스 업데이트
