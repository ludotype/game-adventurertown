# NPC 대사 작성 워크플로우

## 개요
이 워크플로우는 게임 내 NPC(Non-Player Character)의 대사(dialogue)를 체계적으로 작성하기 위한 표준 절차입니다.

## 사용 시기
- 새로운 NPC 캐릭터의 대사 작성 시
- 기존 NPC 대사 수정 또는 추가 시
- 퀘스트/이벤트 관련 대사 작성 시

## 단계별 절차

### 1. 캐릭터 설정 확인
- [ ] NPC의 배경 스토리 확인
- [ ] 성격, 말투, 어조 정의
- [ ] NPC의 역할(상인, 퀘스트 NPC, 배경 NPC 등) 파앀
- [ ] 대사의 목적(정보 제공, 분위기 조성, 유머 등) 명시

### 2. 대사 설계
- [ ] 상황별 대사 분류 (인사, 퀘스트, 특수 이벤트 등)
- [ ] 분기 조건 정의 (플레이어 선택지, 시간, 진행도 등)
- [ ] 대사 길이 적절성 확인 (과도하게 길지 않도록)
- [ ] 반복 노출 시 변형 대사(Variation) 필요 여부 확인

### 3. 초안 작성
- [ ] 1인칭/3인칭 시점 일관성 유지
- [ ] 캐릭터 말투(사투리, 존댓말/반말, 직업 용어 등) 적용
- [ ] 문맥상 자연스러운 대화 흐름 구성
- [ ] 플레이어 선택지와의 연결성 확보

### 4. 검토 및 수정
- [ ] 맞춤법 및 문법 검사
- [ ] 캐릭터 설정과 대사 톤 일치 여부 확인
- [ ] 게임 세계관과의 적합성 검토
- [ ] 중복 표현 또는 클리셰 피하기

### 5. 데이터 포맷팅
- [ ] 프로젝트의 대사 포맷(JSON/YAML/CSV 등) 준수
- [ ] 변수 치환자({playerName}, {itemName} 등) 올바르게 사용
- [ ] ID 네이밍 규칙 준수
- [ ] 주석/메타데이터 추가 (작성자, 버전, 변경 이력)

### 6. 통합 및 테스트
- [ ] 게임 엔진/툴에 데이터 적용
- [ ] 인게임에서 대사 출력 테스트
- [ ] 분기 조건 정상 작동 확인
- [ ] UI 레이아웃과 텍스트 길이 적합성 확인

## 체크리스트
- [ ] 캐릭터 설정과 대사 톤 일치
- [ ] 모든 상황별 대사 작성 완료
- [ ] 맞춤법/문법 오류 없음
- [ ] 데이터 포맷 검증 완료
- [ ] 인게임 테스트 완료
- [ ] 관련 문서(캐릭터 시트 등) 업데이트

## 작성 팁
- **간결함**: 모바일 환경을 고려해 한 화면에 들어올 정도의 길이 권장
- **반복성 피하기**: 같은 의미를 다른 표현으로 변형 작성
- **몰입감**: 게임 세계관에 맞는 용어와 표현 사용
- **접근성**: 너무 어려운 한자어나 전문 용어는 지양
