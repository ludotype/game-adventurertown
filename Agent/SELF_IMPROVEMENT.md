# Self-Improvement & Adaptability

Claude가 이 프로젝트에서 자신의 지침을 확장하고 정리하는 방법입니다.

## 메모리 관리 규칙

### 저장 시점
사용자가 명시적으로 "기억해" / "저장해"라고 하면:

**Pre-flight Check (반드시 먼저 실행)**
1. `memory/MEMORY.md` 읽기 - 기존 메모리 목록 확인
2. 동일/유사 주제 파일 검색
3. 관련 파일 있으면 내용 읽고 비교

**중복 감지 시**
- **완전 중복**: 사용자에게 "이미 동일 내용이 있습니다" 알림
- **부분 중복**: 기존 vs 신규 비교 표시, 선택지 제공
- **타입만 다른 중복** (예: user_ vs feedback_): 통합 제안

**저장 결정 후**
1. 적절한 타입 결정 (`user`, `feedback`, `project`, `reference`)
2. `memory/[타입]_[주제].md` 파일 생성 또는 기존 파일 갱신
3. `memory/MEMORY.md`에 인덱스 추가/갱신

### 정리 규칙
- **memory/ 파일이 10개 이상**: 타입별 폴더(`user/`, `feedback/` 등) 생성 후 이동
- **파일이 30개 이상**: MEMORY.md에 태그/검색 시스템 추가 고려
- **오래된 메모리**: 프로젝트 변경 시 해당 메모리 업데이트 또는 삭제 확인

## 워크플로우 추가 규칙

새로운 작업 유형이 반복될 때 `workflows/`에 문서화:

1. `_template.md` 복사
2. 작업 유형별 규칙, 체크리스트, 예시 포함
3. `CLAUDE.md`의 "작업 유형별 참조" 섹션에 링크 추가

## 파일 정리 (Pruning)

- **3개월 이상 참조되지 않은 파일**: 사용자에게 "삭제 여부" 확인
- **중복 메모리**: 병합 또는 가장 최신 버전만 유지
- **더 이상 관련 없는 reference**: 제거

## 구조 변경 제안

이하 중 하나에 해당하면 구조 변경 제안:
- 특정 폴더의 파일이 20개 이상
- 파일 찾기가 어려워짐
- 새로운 카테고리가 명확히 필요함

**제안 시**: 변경 이유, 새 구조, 마이그레이션 방법을 구체적으로 설명

## 실패 사례 학습

### Case: 파일 무단 덮어쓰기 (2026-04-17)
- **상황**: `CLAUDE.md` 수정 시 `Write`로 통째로 덮어씀
- **결과**: `중요 사항` 섹션 의도치 않게 삭제됨
- **원인**: 
  - 파일 끝부분 확인 안 함
  - `Edit` 대신 `Write` 사용
  - 기존 내용 보존 의식 부재
- **대응**: 
  - CLAUDE.md에 "파일 수정 원칙" 추가
  - `Read` → `Edit` 사용 → 끝까지 확인 습관화
